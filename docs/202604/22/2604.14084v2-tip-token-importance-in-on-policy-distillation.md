## Abstract
On-policy knowledge distillation (OPD) trains a student on its own rollouts under token-level supervision from a teacher. Not all token positions matter equally, but existing views of token importance are incomplete. We ask a direct question: which tokens carry the most useful learning signal in OPD? Our answer is that informative tokens come from two regions: positions with high student entropy, and positions with low student entropy plus high teacher--student divergence, where the student is overconfident and wrong.   Empirically, student entropy is a strong first-order proxy: retaining $50\%$ of tokens with entropy-based sampling matches or exceeds all-token training while reducing peak memory by up to $47\%$. But entropy alone misses a second important region. When we isolate low-entropy, high-divergence tokens, training on fewer than $10\%$ of all tokens nearly matches full-token baselines, showing that overconfident tokens carry dense corrective signal despite being nearly invisible to entropy-only rules.   We organize these findings with TIP (Token Importance in on-Policy distillation), a two-axis taxonomy over student entropy and teacher--student divergence, and give a theoretical explanation for why entropy is useful yet structurally incomplete. This view motivates type-aware token selection rules that combine uncertainty and disagreement. We validate this picture across three teacher--student pairs spanning Qwen3, Llama, and Qwen2.5 on MATH-500 and AIME 2024/2025, and on the DeepPlanning benchmark for long-horizon agentic planning, where Q3-only training on $<$$20\%$ of tokens surpasses full-token OPD. Our experiments are implemented by extending the OPD repository https://github.com/HJSang/OPSD_OnPolicyDistillation, which supports memory-efficient distillation of larger models under limited GPU budgets.

### 1. 论文的核心问题与整体含义（研究动机和背景）
*   **核心问题**：在在线策略知识蒸馏（On-Policy Distillation, OPD）中，学生模型根据自身生成的序列进行学习。然而，并非序列中的每个 Token 对学习的贡献都相同。论文旨在回答：**哪些 Token 携带了最有效的学习信号？**
*   **研究动机**：全量 Token 训练不仅计算开销大，且包含大量冗余信息（如学生已掌握的 Token）。现有的 Token 选择方法（如仅基于熵）是不完整的，容易忽略学生“过度自信且犯错”的关键位置。

### 2. 论文提出的方法论：TIP 分类法
*   **核心思想**：提出 **TIP (Token Importance in on-Policy distillation)**，一个基于**学生熵 ($h_t$)** 和 **师生散度 ($\delta_t$)** 的双轴分类法。
*   **四个象限**：
    *   **Q1 (高熵, 高散度)**：学生不确定且与老师分歧大，携带最密集的纠错信号。
    *   **Q2 (高熵, 低散度)**：学生不确定但与老师一致，有助于稳定预测。
    *   **Q3 (低熵, 高散度)**：**盲点区**。学生非常自信但完全错误（过度自信），携带关键的修正信号。
    *   **Q4 (低熵, 低散度)**：学生自信且正确，属于已掌握知识，信号微弱。
*   **关键技术细节**：
    *   **Soft-OR 分数**：为了同时捕获高熵和高散度 Token，定义无参分数：$s_t = \hat{h}_t + \hat{\delta}_t - \hat{h}_t \cdot \hat{\delta}_t$（其中 $\hat{h}_t, \hat{\delta}_t$ 为归一化值）。
    *   **选择策略**：根据 $s_t$ 分数进行 Top-k 筛选，仅对保留的 Token 计算损失并回传梯度。
*   **理论支撑**：论文证明了仅靠熵无法区分 Q3 和 Q4，而 Soft-OR 能够有效恢复被熵规则遗漏的 Q3 区域。

### 3. 实验设计
*   **数据集与场景**：
    *   **数学推理**：使用 MATH-500、AIME 2024 和 AIME 2025。
    *   **智能体规划**：使用 DeepPlanning 基准（涉及长程旅行规划、购物等复杂约束任务）。
*   **模型对 (Teacher → Student)**：
    *   Qwen3-8B (GRPO) → Qwen3-4B
    *   Llama-3.3-70B → Llama-3.1-8B

*   **对比方法 (Baselines)**：
    *   **Base**: 100% 全量 Token 蒸馏。
    *   **Entropy-only**: 仅基于学生熵进行采样（50%, 20%, 10%）。
    *   **Q3-only**: 仅针对低熵高散度的 Token 进行训练。
    *   **Soft-OR (TIP)**：本文提出的综合筛选方法。

### 4. 资源与算力
*   **硬件设备**：使用了 **NVIDIA H200 GPU**。具体配置为：Qwen3 和 Llama 实验使用 8×H200，Qwen2.5 实验使用 4×H200。
*   **训练细节**：采用 AdamW 优化器，余弦学习率调度。数学任务训练步数未详述，但智能体规划任务（DeepPlanning）统一训练了 15 个 Epoch。
*   **显存优化**：通过 Token 筛选，Qwen3 的峰值显存从 72.0GB 降至约 35.3GB（降幅达 51%），Qwen2.5 从 35.8GB 降至 14.9GB（降幅达 58%）。

### 5. 实验数量与充分性
*   **实验规模**：涵盖了 3 个主流模型家族、4 种模型规模组合、2 大类迥异的任务领域（数学 vs 规划）。
*   **充分性**：
    *   **消融实验**：对比了 Top 50% vs Bottom 50% Token 的效果，验证了低分 Token 确实几乎不含信号。
    *   **盲点验证**：专门设计了 Q3-only 实验，证明了即使只用 <10% 的 Token，只要选对（过度自信处），性能也能接近全量训练。
    *   **客观性**：实验在多个基准上均显示出一致的趋势，且提供了统计标准差，结果较为可信。

### 6. 主要结论与发现
*   **熵是强代理但非万能**：保留 50% 高熵 Token 即可匹配或超越全量训练，同时大幅节省显存。
*   **Q3 区域至关重要**：过度自信的错误（Q3）虽然占比极小（3-15%），但学习效率极高。在 DeepPlanning 任务中，仅用 20% 的 Q3 Token 训练甚至超过了 100% 全量训练。
*   **TIP/Soft-OR 最优**：综合考虑不确定性和分歧度的 Soft-OR 策略在所有数学基准上均优于纯熵策略。

### 7. 优点
*   **高效性**：在保持甚至提升模型性能的前提下，显著降低了训练所需的显存和计算量，使得在有限 GPU 预算下训练更大模型成为可能。
*   **理论与直觉结合**：通过双轴分类法清晰地解释了 Token 重要性的分布，并给出了数学证明。
*   **易于实现**：Soft-OR 分数计算简单，不引入额外的超参数，且利用的是 OPD 流程中现成的中间变量。

### 8. 不足与局限
*   **教师分布依赖**：识别 Q3 区域需要获取教师模型的完整输出分布以计算散度，这在某些黑盒 API 蒸馏场景下不可行。
*   **归一化敏感性**：Soft-OR 依赖 Batch 内的 Min-Max 归一化，可能会受到 Batch 中异常 Token 的影响。
*   **监督形式局限**：实验主要基于逆 KL 散度（Reverse KL），该分类法在正向 KL 或其他损失函数下的表现尚待验证。
