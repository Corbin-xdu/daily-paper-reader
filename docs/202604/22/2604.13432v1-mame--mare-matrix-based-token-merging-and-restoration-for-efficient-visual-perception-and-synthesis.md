## 摘要
Token 压缩对于缓解视觉 Transformer (ViTs) 中自注意力机制的二次复杂度至关重要，因为 ViTs 通常涉及大量输入 token。现有方法（如 ToMe）依赖于 GPU 效率低下的操作（如排序、分散写入），由此产生的额外开销限制了其有效性。我们引入了 MaMe，这是一种完全基于矩阵操作的、无需训练且可微的 token 合并方法，它对 GPU 友好，能够加速 ViTs。此外，我们提出了其逆操作 MaRe 用于 token 恢复，从而构建了用于图像合成的 MaMe+MaRe 流水线。当应用于预训练模型时，MaMe 在准确率仅下降 2% 的情况下使 ViT-B 的吞吐量翻倍。值得注意的是，使用 MaMe 微调最后一层可在 1.1 倍速度下将 ViT-B 的准确率提高 1.0%。在 SigLIP2-B@512 零样本分类中，MaMe 提供了 1.3 倍的加速，且性能下降微乎其微。在视频任务中，MaMe 在 Kinetics-400 上将 VideoMAE-L 加速了 48.5%，而准确率损失仅为 0.84%。此外，MaMe 在某些任务上实现了性能和速度的同时提升。在图像合成中，MaMe+MaRe 流水线在提高质量的同时，将 Stable Diffusion v2.1 的生成延迟降低了 31%。总的来说，这些结果证明了 MaMe 和 MaRe 在加速视觉模型方面的有效性。代码可在 https://github.com/cominder/mame 获取。

## Abstract
Token compression is crucial for mitigating the quadratic complexity of self-attention mechanisms in Vision Transformers (ViTs), which often involve numerous input tokens. Existing methods, such as ToMe, rely on GPU-inefficient operations (e.g., sorting, scattered writes), introducing overheads that limit their effectiveness. We introduce MaMe, a training-free, differentiable token merging method based entirely on matrix operations, which is GPU-friendly to accelerate ViTs. Additionally, we present MaRe, its inverse operation, for token restoration, forming a MaMe+MaRe pipeline for image synthesis. When applied to pre-trained models, MaMe doubles ViT-B throughput with a 2% accuracy drop. Notably, fine-tuning the last layer with MaMe boosts ViT-B accuracy by 1.0% at 1.1x speed. In SigLIP2-B@512 zero-shot classification, MaMe provides 1.3x acceleration with negligible performance degradation. In video tasks, MaMe accelerates VideoMAE-L by 48.5% on Kinetics-400 with only a 0.84% accuracy loss. Furthermore, MaMe achieves simultaneous improvements in both performance and speed on some tasks. In image synthesis, the MaMe+MaRe pipeline enhances quality while reducing Stable Diffusion v2.1 generation latency by 31%. Collectively, these results demonstrate MaMe's and MaRe's effectiveness in accelerating vision models. The code is available at https://github.com/cominder/mame}{https://github.com/cominder/mame.

### 论文分析报告：MaMe & MaRe

#### 1. 论文的核心问题与整体含义
Vision Transformer (ViT) 在视觉任务中表现卓越，但其自注意力机制的计算复杂度随 Token 数量呈二次方增长（$O(N^2)$），这限制了其在高分辨率图像或实时场景中的应用。现有的 Token 压缩方法（如 ToMe）虽然能减少 Token 数量，但往往依赖于排序（Sorting）、散列写入（Scattered Writes）或聚类（Clustering）等操作。这些操作在 GPU 上执行效率较低，产生的额外开销抵消了部分加速效果。

本论文的核心动机是开发一种**完全基于矩阵运算**的 Token 合并（MaMe）与恢复（MaRe）框架。该框架旨在消除非矩阵运算的开销，实现真正的 GPU 加速，同时保持可微性，支持端到端训练或即插即用的推理加速。

#### 2. 论文提出的方法论
论文提出了两个互补的操作：**MaMe**（合并）和 **MaRe**（恢复）。

*   **MaMe (Matrix-Based Token Merging) 核心流程：**
    1.  **Token 分区：** 将输入 Token 序列划分为目标集合（Destination）和源集合（Source）。
    2.  **相似度计算：** 计算两组 Token 之间的余弦相似度矩阵，并使用 ReLU 和偏移阈值 $\tau$ 进行稀疏化，过滤弱连接。
    3.  **自适应权重精炼：** 通过列归一化和动态阈值机制（基于非零权重的平均值）进一步精炼融合权重矩阵 $W_F$。
    4.  **Token 聚合与保留：** 利用 $W_F$ 将源 Token 融合进目标 Token。特别地，如果某个源 Token 与所有目标 Token 都不相似，则将其作为“独特 Token”原样保留，以防信息丢失。
*   **MaRe (Matrix-Based Token Restoration) 核心流程：**
    *   作为 MaMe 的逆过程，利用存储的融合权重矩阵 $W_F$，将合并后的信息按比例重新分配回原始位置，从而恢复原始的 Token 空间结构。
*   **集成方式：** 在感知任务（如分类）中仅使用 MaMe；在生成任务（如 Stable Diffusion）中采用 MaMe + 关注力计算 + MaRe 的流水线，以在降低计算量的同时保持空间分辨率。
