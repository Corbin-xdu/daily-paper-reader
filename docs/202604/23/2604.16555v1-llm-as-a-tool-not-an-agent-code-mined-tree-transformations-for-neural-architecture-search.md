## 摘要
神经架构搜索 (NAS) 旨在自动发现高性能的深度神经网络 (DNN) 架构。然而，传统的算法驱动型 NAS 依赖于精心手工设计的搜索空间以确保可执行性，这限制了开放式探索。最近基于代码的大语言模型 (LLM) 智能体方法虽然减少了人工设计，但目前的 LLM 难以可靠地生成复杂的有效架构，且其提议往往偏向于训练数据中观察到的狭窄模式。为了将可靠的算法搜索与强大的 LLM 辅助相结合，我们提出了 LLMasTool，这是一个基于分层树的 NAS 框架，用于稳定且开放式的模型演化。我们的方法从任意源代码中自动提取可重用模块，并将完整架构表示为分层树，从而通过可靠的树变换而非代码生成来实现演化。在每个演化步骤中，粗粒度规划由多样性引导算法主导，该算法利用贝叶斯建模来提高探索效率，而 LLM 则解决剩余的自由度，以确保有意义的演化轨迹和可执行的生成架构。通过这种形式化方法，我们的方法不再采用完全智能体化的 LLM 方案，而是探索了超出 LLM 固有偏见的各种方向。我们的方法在 CIFAR-10、CIFAR-100 和 ImageNet16-120 上分别比现有 NAS 方法提高了 0.69、1.83 和 2.68 个百分点，证明了其有效性。

## Abstract
Neural Architecture Search (NAS) aims to automatically discover high-performing deep neural network (DNN) architectures. However, conventional algorithm-driven NAS relies on carefully hand-crafted search spaces to ensure executability, which restricts open-ended exploration. Recent coding-based agentic approaches using large language models (LLMs) reduce manual design, but current LLMs struggle to reliably generate complex, valid architectures, and their proposals are often biased toward a narrow set of patterns observed in their training data. To bridge reliable algorithmic search with powerful LLM assistance, we propose LLMasTool, a hierarchical tree-based NAS framework for stable and open-ended model evolution. Our method automatically extracts reusable modules from arbitrary source code and represents full architectures as hierarchical trees, enabling evolution through reliable tree transformations rather than code generation. At each evolution step, coarse-level planning is governed by a diversity-guided algorithm that leverages Bayesian modeling to improve exploration efficiency, while the LLM resolves the remaining degrees of freedom to ensure a meaningful evolutionary trajectory and an executable generated architecture. With this formulation, instead of fully agentic LLM approaches, our method explores diverse directions beyond the inherent biases in the LLM. Our method improves over existing NAS methods by 0.69, 1.83, and 2.68 points on CIFAR-10, CIFAR-100, and ImageNet16-120, demonstrating its effectiveness.

*   **背景挑战**：

### 2. 方法论
*   **核心思想**：通过“代码挖掘”构建模块库，将架构表示为“层级树”，并采用“由粗到细”的决策机制进行演化。
*   **关键技术细节**：
    1.  **算法驱动的模块挖掘**：利用抽象语法树（AST）解析现有的 PyTorch 代码库（如 timm, mmpretrain），自动提取 `nn.Module` 类及其超参数定义，构建可重用的模块数据库 $D_M$。
    2.  **部署友好的树状表示**：将 DNN 架构表示为层级树（类似配置文件），节点代表模块或超参数。这种结构比纯代码更易于算法操作，比图结构更易于人类阅读和部署。
    3.  **树变换演化（Tree Transformations）**：定义了五种基本操作：修改超参数、交换模块、插入/删除模块、创建复合模块、以及“重复上一步”。
    4.  **由粗到细（Coarse-to-Fine）决策流程**：
        *   **粗粒度（算法层）**：使用**层级贝叶斯汤普森采样（Hierarchical Thompson Sampling）**决定演化操作类型和提示词模板，旨在保证探索的多样性并利用历史成功经验。
        *   **细粒度（LLM 层）**：LLM 仅作为工具，在算法确定的位置和约束下，负责填充具体的超参数数值或从候选库中筛选最合适的模块。
*   **算法流程**：从基础架构出发，算法决定“改哪里”和“怎么改”，LLM 决定“改具体成什么”，生成新树后验证可行性，最后进行训练评估并更新历史数据库。

*   **实验规模**：在 4 个主流数据集上进行了完整搜索实验，并针对 ImageNet-100 进行了 1 epoch（快速验证）和 100 epoch（深度验证）的测试。
*   **消融实验**：非常充分，涵盖了模块挖掘的有效性、树表示法的优势、粗到细规划的必要性、汤普森采样的效率、以及对不同 LLM 规模和初始架构的敏感性分析。

### 6. 主要结论与发现
*   **性能卓越**：LLMasTool 在所有数据集上均显著超越了现有 SOTA。在 ImageNet16-120 上比之前的最佳方法 NADER 提高了 **2.68%**。
*   **LLM 并非越强越好**：实验发现 8B 模型的效果甚至略优于 32B，推测是因为更强的模型可能具有更深的先验偏见，而 NAS 需要更多的探索性。
*   **多样性是关键**：通过算法强制进行多样化探索（如“最小化 LLM 决策”模板）比完全依赖 LLM 的“智能”规划效果更好。
*   **历史记忆有效性**：层级汤普森采样能有效利用稀疏的演化历史，显著提升搜索效率。

### 7. 优点
*   **范式创新**：成功将 LLM 从“不可控的生成器”转变为“受控的参数填充工具”，解决了 LLM 在复杂逻辑下的可靠性问题。
*   **资产复用**：通过代码挖掘直接利用人类专家的知识库（如 timm 库），避免了从零开始搜索的低效。
*   **部署友好**：树状表示法直接对应配置文件，搜索到的模型无需复杂的代码重写即可直接使用。

### 8. 不足与局限
*   **系统依赖性**：目前主要针对 PyTorch 和类 OpenMMLab 的配置风格，对于需要自定义 CUDA 内核（如某些 Mamba 变体）的复杂架构集成尚有难度。
*   **时间成本**：虽然比 Agent 方法更稳健，但每个候选架构仍需完整训练，在大规模模型搜索上依然面临巨大的计算开销。
*   **偏差风险**：虽然通过算法增加了多样性，但 LLM 在选择模块超参数时仍可能受到其预训练数据的潜在影响。
