## 摘要
Token 剪枝已成为一种有效的方法，通过丢弃信息量较少的视觉 Token 并保持性能，来减少大视觉语言模型 (LVLMs) 巨大的计算开销。然而，现有方法通常依赖于来自不同 LVLM 组件的单一注意力源，由于注意力分布存在偏差，导致剪枝决策不完整且并非最优。为了解决这一问题，我们提出了 DeSAP，一种新颖的解耦相似度感知剪枝方法，用于在视觉编码器内进行精确的任务感知 Token 剪枝。具体而言，DeSAP 引入了解耦相似度来捕捉视觉特征与文本 Token 之间的细粒度跨模态相关性，为剪枝提供明确的任务相关引导。通过将解耦相似度与源自视觉注意力的视觉显著性信号相结合，DeSAP 在任务相关线索和视觉线索的共同引导下进行 Token 剪枝，即使在激进的剪枝比例下也能实现稳健的剪枝。在不同基准测试和架构上的广泛实验表明，DeSAP 在准确性和效率方面均持续优于现有最先进 (SOTA) 方法。在 LLaVA-1.5-7B 上，DeSAP 通过仅保留 11.1% 的视觉 Token，实现了 10 倍的 FLOPs 减少和 2.3 倍的预填充加速，同时保持了 98.1% 的原始性能。

## Abstract
Token pruning has emerged as an effective approach to reduce the substantial computational overhead of Large Vision-Language Models (LVLMs) by discarding less informative visual tokens while preserving performance. However, existing methods typically rely on individual attention sources from different LVLM components, resulting in incomplete and suboptimal pruning decisions due to biased attention distributions. To address this problem, we propose DeSAP, a novel Decoupled Similarity-Aware Pruning method for precise, task-aware token pruning within the visual encoder. Specifically, DeSAP introduces a decoupled similarity to capture fine-grained cross-modal relevance between visual features and text tokens, providing explicit task-related guidance for pruning. By integrating decoupled similarity with visual saliency signals derived from visual attention, DeSAP performs token pruning under the guidance of both task-related and visual cues, enabling robust pruning even under aggressive pruning ratios. Extensive experiments across diverse benchmarks and architectures show that DeSAP consistently outperforms SOTA methods in both accuracy and efficiency. On LLaVA-1.5-7B, DeSAP achieves a 10 times FLOPs reduction and a 2.3 times prefill speedup by retaining only 11.1% of visual tokens, while maintaining 98.1% of the original performance.

大视觉语言模型（LVLM）在处理高分辨率图像或多帧视频时，会产生海量的视觉 Token，导致巨大的计算开销和推理延迟。现有的 Token 剪枝方法主要分为两类：
*   **视觉中心（Visual-centric）：** 依赖视觉编码器的 [CLS] 注意力，容易偏向全局显著语义而忽略任务相关的局部细节。
*   **任务中心（Task-centric）：** 依赖 LLM 侧的跨模态注意力，但由于 LLM 注意力的空间偏差（倾向于关注序列中邻近的 Token），往往在剪枝初期就丢失了关键信息。
**核心问题：** 如何在视觉编码器端尽早引入准确的任务引导，同时克服注意力分布的偏差，实现高效且无损的 Token 剪枝？

### 2. 方法论：DeSAP (Decoupled Similarity-Aware Pruning)
DeSAP 的核心思想是**解耦视觉特征**，并结合**细粒度跨模态对齐**来生成任务感知的剪枝决策。
*   **特征解耦：** 将视觉编码器最后一层的隐藏状态分解为“解耦残差组件”（捕获全局语义）和“解耦注意力组件”（保留局部细节）。研究发现，去除残差连接后的注意力特征与文本 Token 的对齐度更高。
*   **细粒度跨模态对齐（FCMA）：** 不直接计算余弦相似度，而是通过一种加权机制，衡量视觉 Token 与文本 [EOS] Token、文本 Token 与视觉 [CLS] Token 之间的相互贡献，从而生成更精确的任务相关相似度图。
*   **双源引导剪枝：** 将上述“任务相关相似度”与传统的“[CLS] 显著性注意力”结合。系统按 1:1 的比例分配剪枝预算，既保留了图像本身的显著区域，又保留了与用户提问相关的特定区域。
*   **Token 合并：** 对于未被选中的 Token，通过聚类方式将其信息合并到最相似的保留 Token 中，以减少信息损失。

### 6. 主要结论与发现
*   **解耦是关键：** 视觉编码器的隐藏状态被全局语义主导，通过解耦出的注意力分量能更有效地捕捉局部细节。
*   **双源优于单源：** 仅靠任务引导或仅靠视觉显著性都会导致性能下降，1:1 的平衡分配效果最好。
*   **极端剪枝下的鲁棒性：** 在仅保留 11.1% Token 的极端情况下（88.9% 剪枝率），DeSAP 仍能保持 98.1% 的性能，远超其他方法。

### 7. 优点（亮点）
*   **任务感知前置：** 首次在编码器端实现了细粒度的跨模态任务引导，避免了 LLM 端的空间偏差。
*   **高效性：** 相似度计算的额外开销极小（相对于 LLM 的计算量可忽略不计），且无需任何重新训练。
*   **通用性：** 能够适配多种 LVLM 架构（如 Qwen 和 LLaVA 系列），具有很强的实用价值。

### 8. 不足与局限
*   **指令依赖性：** 当用户提问非常模糊（如“描述这张图”）时，任务引导信号会变弱，模型性能会退化到依赖视觉显著性。
*   **长文本开销：** 虽然视觉 Token 减少了，但如果输入的文本查询极长，计算跨模态相似度的开销会增加（尽管文中引入了文本 Token 筛选策略来缓解）。
*   **动态比例缺失：** 目前任务和视觉的保留比例是固定的（1:1），未来若能根据任务类型动态调整比例可能会进一步提升效果。
