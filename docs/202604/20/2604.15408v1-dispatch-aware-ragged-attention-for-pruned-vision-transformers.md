## 摘要
视觉 Transformer (ViT) 的 Token 剪枝方法通过丢弃无信息的补丁，有望实现注意力机制 FLOPs 的二次方级减少。然而，当使用最先进的变长注意力 API（包括 FlashAttention-2 的 varlen 和 PyTorch 的 NestedTensor SDPA）执行剪枝后的序列时，实际运行延迟（wall-clock latency）并未相应缩减。我们将此归因于调度开销瓶颈：在 ViT 典型的剪枝后短序列长度（<=197 tokens）下，实际矩阵算术运算在个位数微秒内即可完成，而主机端调度路径却消耗了 60-90 微秒。我们提出了一种轻量级的双向 Triton 注意力算子，其调度下限为 40 微秒，比 FlashAttention-2 varlen 低约 1.5 倍，从而使剪枝带来的节省在实际运行时间中更加显著。集成到完整的“打包-注意力-解包”（pack-attend-unpack）流水线后，我们的系统在四种剪枝算法（Threshold-L2、DynamicViT、EViT、ATS）中一致实现了相比填充式 PyTorch SDPA 高达 2.24 倍的端到端吞吐量，支持 DeiT-T/S/B 等不同规模模型，并保持了位精确（bit-exact）的分类预测，最大 Logit 绝对差值小于 0.007。

## Abstract
Token pruning methods for Vision Transformers (ViTs) promise quadratic reductions in attention FLOPs by dropping uninformative patches. Yet when pruned sequences are executed with state-of-the-art variable-length attention APIs -- including FlashAttention-2's varlen and PyTorch's NestedTensor SDPA-the wall-clock attention latency doesn't scale accordingly. We trace this to a dispatch-overhead bottleneck: at the short, post-pruning sequence lengths typical of ViTs (<=197 tokens), actual matrix arithmetic completes in single-digit microseconds while the host-side dispatch path consumes 60-90 us. We present a lightweight, bidirectional Triton attention kernel whose dispatch floor is 40 us roughly 1.5x lower than FlashAttention-2 varlen-allowing pruning savings to become more visible in wall-clock time. Integrated into a complete pack-attend-unpack pipeline, our system achieves up to 2.24x end-to-end throughput over padded PyTorch SDPA consistently across four pruning algorithms (Threshold-L2, DynamicViT, EViT, ATS), scales across DeiT-T/S/B, and maintains bit-exact classification predictions with <0.007 max absolute logit difference.

论文聚焦于**视觉 Transformer (ViT) Token 剪枝技术的实际落地瓶颈**。虽然许多剪枝算法（如 DynamicViT, EViT 等）在理论上能显著减少计算量（FLOPs），但在实际 GPU 推理中，由于以下原因导致加速效果不佳：
*   **填充瓶颈（Padding Bottleneck）**：标准框架需将变长序列填充至统一长度，导致无效计算，甚至比不剪枝更慢。
*   **调度开销瓶颈（Dispatch-Overhead Bottleneck）**：这是本文的核心发现。在 ViT 典型的短序列（≤197 tokens）场景下，现有的变长注意力 API（如 FlashAttention-2 varlen）的主机端调度开销（约 60-90 μs）远超实际算术运算时间（个位数 μs）。这意味着无论剪枝掉多少 Token，延迟都卡在了一个固定的“底噪”上。

### 2. 论文提出的方法论
本文提出了一套针对剪枝 ViT 优化的**调度感知不规则注意力（Dispatch-Aware Ragged Attention）**系统：
*   **核心思想**：通过极简化的内核设计降低主机端调度开销，使短序列的计算节省能够反映在实际运行时间中。

    *   **端到端流水线**：将该内核集成到 ViT 推理中，在剪枝点之后（如 DeiT 的第 5 层起）使用不规则注意力处理打包后的数据，最后提取 CLS Token 进行分类。
