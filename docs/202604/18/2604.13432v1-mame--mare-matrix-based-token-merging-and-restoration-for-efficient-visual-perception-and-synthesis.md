## 摘要
Token 压缩对于缓解视觉 Transformer (ViT) 中自注意力机制的二次复杂度至关重要，因为 ViT 通常涉及大量的输入 Token。现有方法（如 ToMe）依赖于 GPU 效率低下的操作（如排序、离散写入），从而引入了限制其有效性的额外开销。我们提出了 MaMe，一种完全基于矩阵操作的无需训练、可微分的 Token 合并方法，该方法对 GPU 友好，可加速 ViT。此外，我们提出了其逆操作 MaRe 用于 Token 恢复，从而构建了用于图像合成的 MaMe+MaRe 流水线。当应用于预训练模型时，MaMe 在准确率仅下降 2% 的情况下使 ViT-B 的吞吐量翻倍。值得注意的是，使用 MaMe 对最后一层进行微调，在 1.1 倍速度下将 ViT-B 的准确率提升了 1.0%。在 SigLIP2-B@512 零样本分类中，MaMe 提供了 1.3 倍的加速，且性能下降可以忽略不计。在视频任务中，MaMe 在 Kinetics-400 数据集上将 VideoMAE-L 加速了 48.5%，而准确率损失仅为 0.84%。此外，MaMe 在某些任务上实现了性能和速度的双重提升。在图像合成中，MaMe+MaRe 流水线在提高质量的同时，将 Stable Diffusion v2.1 的生成延迟降低了 31%。总之，这些结果证明了 MaMe 和 MaRe 在加速视觉模型方面的有效性。代码可在 https://github.com/cominder/mame 获取。

## Abstract
Token compression is crucial for mitigating the quadratic complexity of self-attention mechanisms in Vision Transformers (ViTs), which often involve numerous input tokens. Existing methods, such as ToMe, rely on GPU-inefficient operations (e.g., sorting, scattered writes), introducing overheads that limit their effectiveness. We introduce MaMe, a training-free, differentiable token merging method based entirely on matrix operations, which is GPU-friendly to accelerate ViTs. Additionally, we present MaRe, its inverse operation, for token restoration, forming a MaMe+MaRe pipeline for image synthesis. When applied to pre-trained models, MaMe doubles ViT-B throughput with a 2% accuracy drop. Notably, fine-tuning the last layer with MaMe boosts ViT-B accuracy by 1.0% at 1.1x speed. In SigLIP2-B@512 zero-shot classification, MaMe provides 1.3x acceleration with negligible performance degradation. In video tasks, MaMe accelerates VideoMAE-L by 48.5% on Kinetics-400 with only a 0.84% accuracy loss. Furthermore, MaMe achieves simultaneous improvements in both performance and speed on some tasks. In image synthesis, the MaMe+MaRe pipeline enhances quality while reducing Stable Diffusion v2.1 generation latency by 31%. Collectively, these results demonstrate MaMe's and MaRe's effectiveness in accelerating vision models. The code is available at https://github.com/cominder/mame}{https://github.com/cominder/mame.

### 1. 论文的核心问题与整体含义
*   **研究背景**：Vision Transformers (ViTs) 在处理高分辨率图像或长视频时，由于自注意力机制的计算复杂度随 Token 数量 $N$ 呈平方增长 ($O(N^2)$)，导致计算开销巨大。
*   **核心问题**：现有的 Token 压缩方法（如 ToMe）虽然能减少 Token 数量，但往往依赖于 **排序（Sorting）**、**散布写入（Scattered Writes）** 或 **聚类（Clustering）** 等操作。这些操作在 GPU 上执行效率低下，引入了额外的计算开销，限制了实际的加速效果。
*   **研究动机**：开发一种完全基于 **矩阵运算** 的 Token 合并（MaMe）与恢复（MaRe）方法，使其既能实现理论上的复杂度降低，又能充分利用 GPU 的并行计算能力，且具备可微性以支持端到端训练。

### 2. 论文提出的方法论
该论文提出了两个互补的模块：**MaMe**（合并）和 **MaRe**（恢复）。

*   **MaMe (Matrix-Based Token Merging) 核心流程**：
    1.  **Token 分区**：将输入 Token 序列划分为目标集合 ($X_{dst}$) 和源集合 ($X_{src}$)，通常采用交替索引的方式。
    2.  **相似度计算**：计算 $X_{dst}$ 与 $X_{src}$ 之间的余弦相似度矩阵，并通过 ReLU 函数和偏移阈值 $\tau$ 过滤弱连接。
    3.  **自适应权重精炼**：通过列归一化和动态阈值剪枝（基于非零权重的平均值），生成最终的融合权重矩阵 $W_F$。
    4.  **聚合与保留**：将源 Token 融合进相似的目标 Token 中。特别地，MaMe 会保留那些与任何目标 Token 都不相似的“独特”源 Token，以防止关键信息丢失。
*   **MaRe (Matrix-Based Token Restoration) 核心流程**：
    *   作为 MaMe 的逆过程，利用存储的融合矩阵 $W_F$，将合并后的 Token 重新分配回原始位置。这对于需要保持空间分辨率的任务（如图像生成）至关重要。
*   **集成方式**：在感知任务中仅使用 MaMe 减少计算量；在生成任务中，在注意力计算前使用 MaMe 压缩，计算后使用 MaRe 恢复，形成“压缩-计算-恢复”的流水线。
