## 摘要
视觉里程计（VO）是嵌入式视觉中的一项关键任务，它通过视觉传感器估计相机运动，是许多嵌入式功耗受限系统（从自主机器人到增强现实和虚拟现实可穿戴设备）的核心组件。最新一类 VO 系统将深度学习模型与受生物启发的事件相机相结合，这类相机对运动模糊和光照条件具有鲁棒性。然而，最先进的（SoA）基于事件的 VO 算法需要大量的内存和计算资源。例如，领先的方法 DEVO 每帧需要 733 MB 内存和 1550 亿次乘加（MAC）运算。我们提出了 TinyDEVO，这是一种专为资源受限的微控制器单元（MCU）设计的基于事件的 VO 深度学习模型。我们将 TinyDEVO 部署在超低功耗（ULP）的 9 核 RISC-V 微控制器上，实现了约 1.2 帧/秒的吞吐量，平均功耗仅为 86 mW。得益于我们的神经网络架构优化和超参数调优，与 DEVO 相比，TinyDEVO 在三个最先进的数据集上将内存占用减少了 11.5 倍（降至 63.8 MB），每帧运算量减少了 29.7 倍（降至每帧 52 亿次 MAC），同时保持了 27 cm 的平均轨迹误差，仅比 DEVO 高出 19 cm。我们的工作首次证明了在超低功耗设备上运行基于事件的 VO 流水线的可行性。

## Abstract
A key task in embedded vision is visual odometry (VO), which estimates camera motion from visual sensors, and it is a core component in many embedded power-constrained systems, from autonomous robots to augmented and virtual reality wearable devices. The newest class of VO systems combines deep learning models with bio-inspired event-based cameras, which are robust to motion blur and lighting conditions. However, state-of-the-art (SoA) event-based VO algorithms require significant memory and computation. For example, the leading approach DEVO requires 733 MB of memory and 155 billion multiply-accumulate (MAC) operations per frame. We present TinyDEVO, an event-based VO deep learning model designed for resource-constrained microcontroller units (MCUs). We deploy TinyDEVO on an ultra-low-power (ULP) 9-core RISC-V-based MCU, achieving a throughput of approximately 1.2 frames per second with an average power consumption of only 86 mW. Thanks to our neural network architectural optimizations and hyperparameter tuning, TinyDEVO reduces the memory footprint by 11.5x (to 63.8 MB) and the number of operations per frame by 29.7x (to 5.2 billion MACs per frame) compared to DEVO, while maintaining an average trajectory error of 27 cm, i.e., only 19 cm higher than DEVO, on three state-of-the-art datasets. Our work demonstrates, for the first time, the feasibility of an event-based VO pipeline on ultra-low-power devices.

    *   **事件相机优势**：相比传统相机，事件相机具有高动态范围、低延迟和无运动模糊的特点，非常适合机器人导航。

    *   **应用需求**：微型无人机、智能眼镜等嵌入式设备急需一种低功耗、轻量化的VO方案。

### 2. 方法论：TinyDEVO 核心思想与技术细节
TinyDEVO 基于 DEVO 架构进行了深度优化，核心思想是**“模型剪裁 + 架构重构 + 超参数精细调优”**：
*   **Patchifier（特征提取器）优化**：
    *   **通道缩减**：大幅减少匹配特征（MF）和上下文特征（CF）的通道数（如从128/384降至64/96）。
    *   **移除旁路连接**：去除CNN中的残差连接以简化计算。
    *   **移除金字塔特征（PYR）**：舍弃较小尺度的特征图以节省内存。
*   **Update Block（更新块）重构**：
    *   **轻量化GRU**：将复杂的门控循环单元（GRU）替换为简单的归一化层加全连接层。
    *   **移除Softmax聚合（SA）**：在某些配置中移除计算密集型的Softmax操作，以减少FPU负担。
*   **Patch Graph（补丁图）优化**：
    *   通过调整补丁数量（$N_{patches}$）、移除窗口（$R_w$）和补丁寿命（$PLT$），显著减少图中的边数（$N_{edges}$），因为计算量与边数成正比。
*   **混合精度部署**：在 GAP9 MCU 上，将深度学习模块量化为 **int8**，将几何计算模块（相关块和束调整）保持为 **FP16/BF16**。

### 6. 主要结论与发现
*   **可行性突破**：首次证明了在亚100mW功耗的MCU上运行深度学习事件VO是可行的。
*   **极致压缩**：TinyDEVO 相比 DEVO 实现了 **11.5倍的内存缩减**（63.8 MB）和 **29.7倍的运算量缩减**（5.2 GMAC/帧）。
*   **精度权衡**：虽然精度有所下降（平均误差增加约19cm），但在资源受限场景下，其 ATE（如MVSEC上27cm）仍处于可接受范围，且在某些数据集上优于传统的几何方法（如EVO）。

### 7. 优点与亮点
*   **软硬件协同**：不仅优化了算法，还针对 RISC-V 架构的 NE16 加速器和多核特性进行了深度适配。
*   **帕累托优化**：通过几何方法寻找“膝点（Knee Point）”，在精度和速度之间找到了最优的平衡配置。
*   **超越几何方法**：在内存占用远小于传统几何方法（如 ORB-SLAM3 和 EVO）的同时，实现了更好的精度-内存权衡。

### 8. 不足与局限
*   **实时性尚存差距**：1.2 FPS 的速度对于高速飞行的无人机来说仍然偏慢（通常需要 >10 FPS），目前仅能满足“软实时”或低速移动需求。
*   **精度损失**：相比原版 DEVO，在长距离轨迹上的累计误差增长较快。
*   **外部内存依赖**：虽然是 MCU 方案，但仍需 8MB 以上的外部 HyperRAM，这增加了系统设计的复杂性和部分功耗。
*   **场景限制**：实验主要集中在室内环境，室外高速或极端复杂场景的鲁棒性有待进一步验证。
