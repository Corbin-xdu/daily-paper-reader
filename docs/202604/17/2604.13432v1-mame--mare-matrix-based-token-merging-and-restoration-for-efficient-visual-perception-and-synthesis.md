## 摘要
Token 压缩对于缓解视觉 Transformer (ViT) 中自注意力机制的二次复杂度至关重要，因为 ViT 通常涉及大量的输入 Token。现有方法（如 ToMe）依赖于 GPU 效率低下的操作（如排序、离散写入），从而引入了限制其有效性的额外开销。我们提出了 MaMe，这是一种完全基于矩阵操作的、无需训练且可微的 Token 合并方法，它对 GPU 友好，可加速 ViT。此外，我们提出了其逆操作 MaRe 用于 Token 恢复，从而构建了一个用于图像合成的 MaMe+MaRe 流水线。当应用于预训练模型时，MaMe 在准确率仅下降 2% 的情况下使 ViT-B 的吞吐量翻倍。值得注意的是，使用 MaMe 对最后一层进行微调，在 1.1 倍速度下将 ViT-B 的准确率提升了 1.0%。在 SigLIP2-B@512 零样本分类中，MaMe 提供了 1.3 倍的加速，且性能下降可以忽略不计。在视频任务中，MaMe 在 Kinetics-400 数据集上将 VideoMAE-L 加速了 48.5%，而准确率损失仅为 0.84%。此外，MaMe 在某些任务上实现了性能和速度的双重提升。在图像合成中，MaMe+MaRe 流水线在提高质量的同时，将 Stable Diffusion v2.1 的生成延迟降低了 31%。总之，这些结果证明了 MaMe 和 MaRe 在加速视觉模型方面的有效性。代码可在 https://github.com/cominder/mame 获取。

## Abstract
Token compression is crucial for mitigating the quadratic complexity of self-attention mechanisms in Vision Transformers (ViTs), which often involve numerous input tokens. Existing methods, such as ToMe, rely on GPU-inefficient operations (e.g., sorting, scattered writes), introducing overheads that limit their effectiveness. We introduce MaMe, a training-free, differentiable token merging method based entirely on matrix operations, which is GPU-friendly to accelerate ViTs. Additionally, we present MaRe, its inverse operation, for token restoration, forming a MaMe+MaRe pipeline for image synthesis. When applied to pre-trained models, MaMe doubles ViT-B throughput with a 2% accuracy drop. Notably, fine-tuning the last layer with MaMe boosts ViT-B accuracy by 1.0% at 1.1x speed. In SigLIP2-B@512 zero-shot classification, MaMe provides 1.3x acceleration with negligible performance degradation. In video tasks, MaMe accelerates VideoMAE-L by 48.5% on Kinetics-400 with only a 0.84% accuracy loss. Furthermore, MaMe achieves simultaneous improvements in both performance and speed on some tasks. In image synthesis, the MaMe+MaRe pipeline enhances quality while reducing Stable Diffusion v2.1 generation latency by 31%. Collectively, these results demonstrate MaMe's and MaRe's effectiveness in accelerating vision models. The code is available at https://github.com/cominder/mame}{https://github.com/cominder/mame.

### 1. 核心问题与研究动机
*   **核心问题**：Vision Transformers (ViTs) 的自注意力机制具有 $O(N^2)$ 的计算复杂度，在处理高分辨率图像或长视频时计算开销巨大。
*   **研究动机**：现有的 Token 减少技术（如 ToMe）虽然能减少 Token 数量，但往往依赖于**排序（Sorting）**、**离散写入（Scattered writes）**或**聚类（Clustering）**等操作。这些操作在 GPU 上执行效率低下，产生的额外开销（Overhead）抵消了 Token 减少带来的加速效果。
*   **目标**：开发一种完全基于**矩阵运算**、GPU 友好、可微分且无需额外参数的 Token 合并与恢复框架。

### 2. 方法论：MaMe 与 MaRe
该论文提出了两个互补的操作：

*   **MaMe (Matrix-Based Token Merging - 合并)**：
    1.  **Token 分区**：将输入 Token 序列划分为“目标 Token (Destination)”和“源 Token (Source)”两个不相交的集合（通常采用交替索引方式）。
    2.  **相似度计算**：计算两组 Token 之间的余弦相似度矩阵。
    3.  **自适应权重细化**：通过 ReLU 激活函数和动态阈值（$\tau$）过滤弱连接，生成稀疏的融合矩阵。
    4.  **Token 聚合与保留**：将相似的源 Token 融合进目标 Token，同时通过掩码机制保留那些与任何目标 Token 都不相似的“独特”源 Token，以防信息丢失。
*   **MaRe (Matrix-Based Token Restoration - 恢复)**：
    *   这是 MaMe 的逆过程。利用合并阶段存储的融合矩阵，通过矩阵乘法将信息重新分配回原始的 Token 位置。这对于需要保持空间分辨率的图像生成任务（如 Stable Diffusion）至关重要。
*   **技术特点**：完全基于矩阵乘法，能够无缝集成 Flash Attention 等硬件加速算子，且支持端到端训练。

*   **硬件友好**：避开了 GPU 厌恶的非连续内存访问和排序操作，真正实现了理论加速向实际加速的转化。

*   **多功能性**：不仅适用于感知任务，通过 MaRe 也能完美适配生成任务。

*   **超参数敏感性**：相似度阈值 $\tau$ 需要根据不同模型和任务手动调整，缺乏一种自动化的启发式搜索方法。

*   **极端压缩下的风险**：在极高压缩率下，虽然矩阵运算依然快，但语义信息的丢失仍不可避免（尤其在小模型 ViT-S 上表现明显）。
