## 摘要
Token 剪枝已成为一种有效的方法，通过丢弃信息量较少的视觉 Token 并保持性能，来减少大视觉语言模型 (LVLMs) 巨大的计算开销。然而，现有方法通常依赖于来自不同 LVLM 组件的单一注意力源，由于注意力分布存在偏差，导致剪枝决策不完整且并非最优。为了解决这一问题，我们提出了 DeSAP，一种新颖的解耦相似度感知剪枝方法，用于在视觉编码器内进行精确的任务感知 Token 剪枝。具体而言，DeSAP 引入了解耦相似度来捕捉视觉特征与文本 Token 之间的细粒度跨模态相关性，为剪枝提供明确的任务相关引导。通过将解耦相似度与源自视觉注意力的视觉显著性信号相结合，DeSAP 在任务相关线索和视觉线索的共同引导下进行 Token 剪枝，即使在激进的剪枝比例下也能实现稳健的剪枝。在不同基准测试和架构上的广泛实验表明，DeSAP 在准确性和效率方面均持续优于 SOTA 方法。在 LLaVA-1.5-7B 上，DeSAP 通过仅保留 11.1% 的视觉 Token，实现了 10 倍的 FLOPs 减少和 2.3 倍的预填充加速，同时保持了 98.1% 的原始性能。

## Abstract
Token pruning has emerged as an effective approach to reduce the substantial computational overhead of Large Vision-Language Models (LVLMs) by discarding less informative visual tokens while preserving performance. However, existing methods typically rely on individual attention sources from different LVLM components, resulting in incomplete and suboptimal pruning decisions due to biased attention distributions. To address this problem, we propose DeSAP, a novel Decoupled Similarity-Aware Pruning method for precise, task-aware token pruning within the visual encoder. Specifically, DeSAP introduces a decoupled similarity to capture fine-grained cross-modal relevance between visual features and text tokens, providing explicit task-related guidance for pruning. By integrating decoupled similarity with visual saliency signals derived from visual attention, DeSAP performs token pruning under the guidance of both task-related and visual cues, enabling robust pruning even under aggressive pruning ratios. Extensive experiments across diverse benchmarks and architectures show that DeSAP consistently outperforms SOTA methods in both accuracy and efficiency. On LLaVA-1.5-7B, DeSAP achieves a 10 times FLOPs reduction and a 2.3 times prefill speedup by retaining only 11.1% of visual tokens, while maintaining 98.1% of the original performance.

### 1. 核心问题与整体含义（研究动机和背景）
大视觉语言模型（LVLMs）在处理高分辨率图像或多帧视频时，会产生海量的视觉 Token，导致巨大的计算开销和推理延迟。现有的 Token 剪枝方法主要分为两类：
*   **以视觉为中心（Visual-centric）：** 依赖视觉编码器的 `[CLS]` 注意力，容易偏向全局显著区域，忽略任务相关的局部细节。
*   **以任务为中心（Task-centric）：** 依赖 LLM 侧的跨模态注意力，但由于 LLM 的注意力存在空间偏差（倾向于关注序列末尾或附近的 Token），且引入任务引导过晚，导致关键信息在进入 LLM 前就已丢失。
**核心问题：** 如何在视觉编码器阶段就引入精确的任务感知引导，以克服注意力偏差并实现高效、无损的剪枝？

### 2. 方法论：DeSAP (Decoupled Similarity-Aware Pruning)
DeSAP 的核心思想是**解耦视觉特征**，提取能保留局部细节的分量，并与文本进行**细粒度跨模态对齐**。

*   **特征解耦（Hidden State Decomposition）：** 将视觉编码器最后一层的隐藏状态分解为“残差分量”（捕获全局语义）和“解耦注意力分量”（保留局部细节）。研究发现，解耦后的注意力分量（$A_d$）与文本的对齐度更高。
*   **细粒度跨模态对齐（FCMA）：** 不直接计算余弦相似度，而是通过 Token 级别的加权贡献来分解相似度矩阵。利用视觉的 `[CLS]` 和文本的 `[EOS]` 作为全局代理，计算每个 Token 的重要性权重，从而生成更精确的任务相关相似度图（$A_t$）。
*   **双源引导剪枝：** 结合任务相关相似度图（$A_t$）和传统的视觉显著性图（$A_s$，来自 `[CLS]` 注意力），按比例（默认 1:1）分配剪枝预算。
*   **Token 合并：** 对于未被选中的 Token，不直接丢弃，而是通过聚类方式将其信息合并到最相似的保留 Token 中，以减少信息损失。

### 6. 主要结论与发现
*   **性能卓越：** 在 LLaVA-1.5-7B 上，仅保留 11.1% 的 Token（剪枝率 88.9%），仍能保持 98.1% 的原始性能，且在 192 Token 设置下甚至超过了全量模型。
*   **效率提升：** 实现了 **10 倍的 FLOPs 减少**，在预填充阶段获得 **2.3 倍的加速**。
*   **解耦的价值：** 实验证明，从残差中分离出注意力分量是获取细粒度任务引导的关键，这比单纯修正 `[CLS]` 偏差更有效。

### 7. 优点
*   **任务感知前置：** 创新性地在编码器端实现了细粒度的任务引导，避免了后期剪枝的信息滞后问题。
*   **无需训练：** 作为一种后处理方法，具有极强的通用性和低部署成本。
*   **鲁棒性强：** 在极端剪枝比例（如 90% 左右）下，性能下降远小于同类方法，尤其在处理图像中的小物体时表现优异。

### 8. 不足与局限
*   **文本依赖性：** 当用户查询（Query）非常模糊或不包含明确语义（如“描述这张图”）时，任务引导的效果会减弱，模型会退化为依赖视觉显著性。
*   **长文本开销：** 虽然论文引入了文本 Token 筛选策略，但如果输入文本极长，计算跨模态相似度的开销仍会增加。
*   **动态比例：** 目前任务引导与视觉引导的比例是固定分配（1:1），未来若能根据任务类型动态调整比例可能会进一步提升效果。
