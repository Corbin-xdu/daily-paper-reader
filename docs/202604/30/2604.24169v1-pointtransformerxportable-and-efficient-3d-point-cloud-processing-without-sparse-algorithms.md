## 摘要
3D 点云感知仍然紧密耦合于用于空间操作的自定义 CUDA 算子，这限制了其在非 NVIDIA、AMD 以及嵌入式硬件上的可移植性和效率。我们推出了 PointTransformerX (PTX)，这是一个完全基于 PyTorch 原生的 3D 点云视觉 Transformer 骨干网络，在移除所有自定义 CUDA 算子和外部库的同时，保持了极具竞争力的准确率。PTX 引入了 3D-GS-RoPE，这是一种旋转位置嵌入，可在不构建邻域的情况下直接在自注意力机制中编码 3D 空间关系，并进一步用线性投影取代了稀疏卷积补丁嵌入。PTX 探索了推理时注意力窗口的缩放，以在无需重新训练的情况下提高准确率。通过重新设计的前馈网络，PTX 在 ScanNet 数据集上达到了 PointTransformer V3 98.7% 的准确率，参数量减少了 79.2%，执行速度快了 1.6 倍，且仅需 253 MB 内存。PTX 可在 NVIDIA GPU、AMD GPU (ROCm) 和 CPU 上原生运行，为点云感知提供了一个高效且可移植的基础。

## Abstract
3D point cloud perception remains tightly coupled to custom CUDA operators for spatial operations, limiting portability and efficiency on non-NVIDIA, AMD, and embedded hardware. We introduce PointTransformerX (PTX), a fully PyTorch-native vision transformer backbone for 3D point clouds, removing all custom CUDA operators and external libraries while retaining competitive accuracy. PTX introduces 3D-GS-RoPE, a rotary positional embedding that encodes 3D spatial relationships directly in self-attention without neighborhood construction, and further replaces sparse convolutional patch embedding with a linear projection. PTX explores inference-time scaling of attention windows to improve accuracy without retraining. With a redesigned feed-forward network, PTX achieves 98.7\% of PointTransformer V3's accuracy on ScanNet with 79.2\% fewer parameters and executing 1.6\times faster while requiring just 253 MB memory. PTX runs natively on NVIDIA GPUs, AMD GPUs (ROCm), and CPUs, providing an efficient and portable foundation for point cloud perception.

### 论文总结：PointTransformerX (PTX)

#### 1. 核心问题与整体含义
**研究动机与背景：**
当前的 3D 点云感知模型（如 PointTransformer 系列、MinkUNet 等）高度依赖于自定义的 CUDA 算子和特定的稀疏算法库（如 `spconv`、`MinkowskiEngine`）。这种现状导致了三个主要问题：
*   **可移植性差：** 难以在非 NVIDIA 硬件（如 AMD GPU、嵌入式设备、CPU）上部署。
*   **维护困难：** 碎片化的第三方库往往维护滞后，难以兼容最新的 PyTorch 特性和硬件架构（如 Hopper 架构的 FP8 支持）。
*   **效率瓶颈：** 自定义算子往往无法充分利用 PyTorch 原生的内核融合（Kernel Fusion）和 JIT 编译优化。

**核心目标：** 提出 PointTransformerX (PTX)，构建一个**完全基于 PyTorch 原生算子**的 3D Transformer 骨干网络，在彻底摆脱自定义 CUDA 依赖的同时，保持与 SOTA 模型相当的精度和更高的效率。

#### 2. 方法论
PTX 基于 PointTransformer V3 (PTv3) 的序列化架构，但进行了彻底的“去稀疏化”改造：

*   **3D-GS-RoPE (3D Gram-Schmidt Rotary Position Embedding)：**
    *   **核心思想：** 针对点云的 3D 特性改进 RoPE。传统的 Axial-RoPE 独立处理各轴，缺乏跨轴敏感度（无法区分对角线位置）；Mixed-RoPE 则因缺乏约束容易导致轴向失衡。
    *   **技术细节：** PTX 为每个注意力头学习一个正交变换矩阵。通过 **Gram-Schmidt 正交化过程** 约束旋转矩阵，确保在旋转坐标系时保持点与点之间的欧几里得距离和角度。这使得模型能以极小的参数代价（每头仅需 6 个可学习参数）捕获复杂的 3D 空间关系。
*   **线性补丁嵌入 (Linear Patch Embedding)：** 放弃了 PTv3 中使用的稀疏卷积层，改用标准的线性投影层进行初始特征提取，实现了全原生算子化。
*   **推理时窗口缩放 (Inference-time Window Scaling)：** 利用 RoPE 的外推属性，PTX 在训练时使用较小的注意力窗口（如 512 个点），而在推理时直接扩展窗口（如 2048 个点）。实验发现这种缩放能显著提升精度，且无需重新训练。
*   **高效前馈网络 (FFN) 重构：** 评估了多种激活函数（GeLU, SwiGLU, ReLU²）和扩展比。最终采用 **ReLU² 激活函数** 并将扩展比从 4 降至 2，在减少参数量的同时缓解了在小规模点云数据集上的过拟合问题。

#### 3. 实验设计
*   **数据集：** 主要在 **ScanNet**（室内场景语义分割）上进行验证。
*   **Benchmark：** 以 **PointTransformer V3 (PTv3)** 为主要基准。
*   **对比方法：** 涵盖了点云处理的三大流派：
    *   **Transformer 类：** PTv1/v2/v3, Stratified Transformer, Swin3D, LitePT。
    *   **卷积/稀疏类：** MinkUNet, SpUNet, OACNN。
    *   **混合类：** OctFormer。
*   **评估指标：** mIoU（精度）、参数量、推理延迟（ms）、峰值内存占用（MB/GB）。

#### 4. 资源与算力
*   **训练硬件：** 使用单张 **NVIDIA H100** GPU。
*   **训练配置：** Batch size 为 12，采用 AdamW 优化器，混合使用 CrossEntropyLoss 和 LovaszLoss。
*   **推理测试平台：** 为了验证可移植性，作者在多种硬件上进行了测试：
    *   NVIDIA H100 (Hopper 架构)
    *   AMD Instinct MI300X (ROCm 平台)
    *   NVIDIA Jetson Orin AGX (嵌入式)
    *   Intel i5-10400F (CPU)
*   **精度模式：** 全程使用 **BF16**，并在 H100 上对部分线性层应用了 **FP8**。

#### 5. 实验数量与充分性
*   **实验规模：** 论文进行了详尽的消融实验，包括：

*   **充分性评价：** 实验设计非常充分且客观。作者不仅对比了精度，还深入分析了内存和延迟，并跨越了四种完全不同的硬件架构，有力证明了其“全原生”带来的可移植性优势。

#### 6. 主要结论与发现
*   **高性能与轻量化：** PTX 仅用 PTv3 **20.8% 的参数量**（9.6M），达到了其 **98.7% 的精度**（ScanNet mIoU 0.765）。
*   **高效率：** 在 H100 上比 PTv3 快 1.6 倍，峰值内存仅为 253 MB。
*   **真正的可移植性：** PTX 是唯一一个无需修改代码即可在 AMD GPU 和 CPU 上高效运行的高性能模型。相比之下，OctFormer 的 CPU 回退模式内存占用极高且速度极慢。
*   **RoPE 的潜力：** 证明了在点云领域，通过合理的几何约束（Gram-Schmidt），RoPE 可以完全取代稀疏卷积的归纳偏置。

#### 7. 优点
*   **架构纯粹：** 彻底去除了对 `spconv` 等复杂第三方库的依赖，极大降低了工程落地难度。
*   **硬件友好：** 支持 BF16 和 FP8，能无缝利用现代 GPU 的加速特性。
*   **创新性位置编码：** 3D-GS-RoPE 解决了多维位置编码中的跨轴敏感性问题，具有较强的理论和实用价值。
*   **推理灵活性：** 窗口缩放技术提供了一种无需重训即可平衡精度与速度的手段。

#### 8. 不足与局限
*   **精度微降：** 尽管非常接近，但 PTX 的绝对精度（mIoU）仍略低于 PTv3（0.765 vs 0.775），这表明稀疏卷积的某些归纳偏置在极端精度要求下仍有价值。
*   **系统瓶颈转移：** 移除稀疏算子后，点云的**序列化排序（Sorting）和池化（Pooling）**成为了新的 CPU/GPU 同步瓶颈，这限制了端到端速度的进一步飞跃。
*   **任务覆盖：** 实验主要集中在语义分割任务上，对于目标检测、实例分割等更复杂任务的泛化能力尚待进一步验证。
