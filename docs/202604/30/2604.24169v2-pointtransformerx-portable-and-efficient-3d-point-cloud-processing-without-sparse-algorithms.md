## 摘要
3D 点云感知目前仍与用于空间操作的自定义 CUDA 算子紧密耦合，这限制了其在非 NVIDIA、AMD 以及嵌入式硬件上的可移植性和效率。我们推出了 PointTransformerX (PTX)，这是一个完全基于 PyTorch 原生的 3D 点云视觉 Transformer 骨干网络，在移除所有自定义 CUDA 算子和外部库的同时，保持了具有竞争力的准确率。PTX 引入了 3D-GS-RoPE，这是一种旋转位置嵌入，可在不构建邻域的情况下直接在自注意力机制中编码 3D 空间关系，并进一步用线性投影取代了稀疏卷积补丁嵌入。PTX 探索了推理时注意力窗口的缩放，以在无需重新训练的情况下提高准确率。通过重新设计的前馈网络，PTX 在 ScanNet 数据集上达到了 PointTransformer V3 98.7% 的准确率，参数量减少了 79.2%，运行速度提高了 1.6 倍，且仅需 253 MB 内存。PTX 可在 NVIDIA GPU、AMD GPU (ROCm) 和 CPU 上原生运行，为点云感知提供了一个高效且可移植的基础。

## Abstract
3D point cloud perception remains tightly coupled to custom CUDA operators for spatial operations, limiting portability and efficiency on non-NVIDIA, AMD, and embedded hardware. We introduce PointTransformerX (PTX), a fully PyTorch-native vision transformer backbone for 3D point clouds, removing all custom CUDA operators and external libraries while retaining competitive accuracy. PTX introduces 3D-GS-RoPE, a rotary positional embedding that encodes 3D spatial relationships directly in self-attention without neighborhood construction, and further replaces sparse convolutional patch embedding with a linear projection. PTX explores inference-time scaling of attention windows to improve accuracy without retraining. With a redesigned feed-forward network, PTX achieves 98.7\% of PointTransformer V3's accuracy on ScanNet with 79.2\% fewer parameters and executing 1.6\times faster while requiring just 253 MB memory. PTX runs natively on NVIDIA GPUs, AMD GPUs (ROCm), and CPUs, providing an efficient and portable foundation for point cloud perception.

*   **核心问题**：当前的 3D 点云感知模型（如 PointTransformer V3, MinkUNet 等）高度依赖于**自定义 CUDA 算子**（如稀疏卷积、kNN 搜索、半径搜索）。

*   **整体含义**：PTX 旨在构建一个**完全基于 PyTorch 原生算子**的 Transformer 骨干网络，通过消除所有稀疏算法和自定义算子，实现 3D 点云处理的高可移植性和极致效率。

### 2. 提出的方法论
PTX 基于序列化架构（如 PTv3），但通过以下关键技术彻底去除了对稀疏算法的依赖：
*   **3D-GS-RoPE（3D Gram-Schmidt 旋转位置嵌入）**：
    *   **核心思想**：传统的 Axial-RoPE（轴向旋转位置嵌入）缺乏跨轴敏感性。PTX 引入了学习型的正交坐标旋转。
    *   **技术细节**：每个注意力头学习一个正交变换矩阵。通过 **Gram-Schmidt 正交化过程**约束旋转矩阵，确保在旋转坐标系时保持点与点之间的欧几里得距离和角度关系。这使得模型无需构建邻域矩阵即可在自注意力中直接编码 3D 空间关系。
*   **线性补丁嵌入（Linear Patch Embedding）**：用标准的线性投影层取代了 PTv3 中极其耗费内存和依赖特定库的稀疏卷积层。
*   **推理时窗口缩放（Inference-time Window Scaling）**：借鉴大语言模型（LLM）的经验，发现 RoPE 具有良好的外推性。PTX 在训练时使用较小的注意力窗口（如 512），在推理时扩展到更大窗口（如 2048），在不增加训练成本的情况下显著提升精度。
*   **高效前馈网络（Efficient FFN）**：重新评估了 FFN 设计，采用 **ReLU² 激活函数**并将扩展率从 4 降低到 2，在减少参数量的同时避免了在小规模点云数据集上的过拟合。

*   **创新的位置编码**：3D-GS-RoPE 巧妙地解决了 Transformer 处理 3D 几何关系时的坐标敏感性问题。
*   **极致的轻量化**：在保持高性能的同时，将参数量压缩到了 10M 以下，非常适合移动端部署。
*   **现代特性支持**：由于是原生 PyTorch 实现，能够直接受益于 FP8、FlashAttention-3 等最新硬件加速技术。
