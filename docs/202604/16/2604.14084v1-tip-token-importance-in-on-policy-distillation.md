## 摘要
同策略知识蒸馏（OPD）在教师模型的 Token 级监督下，利用学生模型自身的生成序列（rollouts）对其进行训练。并非所有的 Token 位置都同样重要，但现有关于 Token 重要性的观点尚不完整。我们提出了一个直接的问题：在 OPD 中，哪些 Token 携带了最有用的学习信号？我们的回答是，信息丰富的 Token 来自两个区域：学生模型熵值较高的位置，以及学生模型熵值较低但教师与学生散度（divergence）较高的位置（即学生模型过度自信且错误的位置）。实验表明，学生模型的熵是一个强有力的一阶代理指标：通过基于熵的采样保留 50% 的 Token，其效果可媲美或超过全量 Token 训练，同时减少高达 47% 的峰值显存。但仅靠熵会遗漏第二个重要区域。当我们分离出低熵、高散度的 Token 时，仅在不足 10% 的 Token 上进行训练就几乎达到了全量 Token 基准的效果，这表明过度自信的 Token 携带了密集的纠错信号，尽管它们在仅基于熵的规则下几乎不可见。我们通过 TIP（同策略蒸馏中的 Token 重要性）整理了这些发现，这是一种基于学生模型熵和师生散度的双轴分类法，并从理论上解释了为什么熵虽然有用但在结构上是不完整的。这一观点启发了结合不确定性（uncertainty）和分歧（disagreement）的类型感知 Token 选择规则。我们在涵盖 Qwen3、Llama 和 Qwen2.5 的三组教师-学生对上验证了这一方案，测试集包括 MATH-500、AIME 2024/2025 以及用于长程智能体规划的 DeepPlanning 基准测试。在 DeepPlanning 中，仅在不足 20% 的 Token 上进行训练就超过了全量 Token 的 OPD。我们的实验通过扩展 OPD 仓库实现，该仓库支持在有限的 GPU 预算下对大型模型进行显存高效的蒸馏。

## Abstract
On-policy knowledge distillation (OPD) trains a student on its own rollouts under token-level supervision from a teacher. Not all token positions matter equally, but existing views of token importance are incomplete. We ask a direct question: which tokens carry the most useful learning signal in OPD? Our answer is that informative tokens come from two regions: positions with high student entropy, and positions with low student entropy plus high teacher--student divergence, where the student is overconfident and wrong.   Empirically, student entropy is a strong first-order proxy: retaining $50\%$ of tokens with entropy-based sampling matches or exceeds all-token training while reducing peak memory by up to $47\%$. But entropy alone misses a second important region. When we isolate low-entropy, high-divergence tokens, training on fewer than $10\%$ of all tokens nearly matches full-token baselines, showing that overconfident tokens carry dense corrective signal despite being nearly invisible to entropy-only rules.   We organize these findings with TIP (Token Importance in on-Policy distillation), a two-axis taxonomy over student entropy and teacher--student divergence, and give a theoretical explanation for why entropy is useful yet structurally incomplete. This view motivates type-aware token selection rules that combine uncertainty and disagreement. We validate this picture across three teacher--student pairs spanning Qwen3, Llama, and Qwen2.5 on MATH-500 and AIME 2024/2025, and on the DeepPlanning benchmark for long-horizon agentic planning, where Q3-only training on $<$$20\%$ of tokens surpasses full-token OPD. Our experiments are implemented by extending the OPD repository https://github.com/HJSang/OPSD_OnPolicyDistillation, which supports memory-efficient distillation of larger models under limited GPU budgets.

### 1. 核心问题与研究动机
*   **核心问题**：在在线策略蒸馏（On-Policy Distillation, OPD）中，学生模型根据自身生成的序列进行学习，但并非所有 Token 都能提供同等的学习价值。论文旨在回答：**哪些 Token 携带了最有用的学习信号？**
*   **研究背景**：传统的 OPD 训练对所有 Token 均匀施加损失，这在计算上是低效的。虽然“高熵（不确定性）”通常被视为重要信号，但作者发现仅靠熵会产生结构性盲点，忽略了那些学生模型“过度自信且犯错”的关键位置。

### 2. 方法论：TIP 分类法与 Soft-OR 评分
*   **核心思想**：提出 **TIP (Token Importance in on-Policy distillation)** 分类框架，从**学生熵 ($h_t$)** 和**师生散度 ($\delta_t$)** 两个维度将 Token 划分为四个象限：
    *   **Q1 (高熵, 高散度)**：不确定且错误，提供最密集的纠错信号。
    *   **Q2 (高熵, 低散度)**：不确定但正确，提供稳定预测的信号。
    *   **Q3 (低熵, 高散度)**：**过度自信且错误（盲点）**，携带关键的系统性偏差修正信号。
    *   **Q4 (低熵, 低散度)**：已解决的 Token，几乎不提供学习信号。
*   **关键技术：Soft-OR 评分**：
    *   为了同时捕获高熵区域和过度自信区域，作者提出了一个无参数的评分公式：$s_t = \hat{h}_t + \hat{\delta}_t - \hat{h}_t \cdot \hat{\delta}_t$（其中 $\hat{h}_t, \hat{\delta}_t$ 为归一化后的值）。
    *   该评分确保只要 Token 在任一维度表现突出（高不确定性或高分歧），就会被选中进行训练。
*   **算法流程**：在训练过程中，实时计算每个 Token 的 $s_t$，根据预设的保留比例 $\rho$ 选择 Top-K 个 Token 计算损失并进行反向传播，其余 Token 被屏蔽。

### 6. 主要结论与发现
*   **熵是强代理但非唯一**：保留 50% 高熵 Token 即可匹配全量训练效果，说明 Q4 区域（约 45% 的 Token）确实是冗余的。
*   **过度自信是关键盲点**：Q3 区域虽然 Token 数量极少（3%-15%），但包含极高价值的修正信号。在规划任务中，仅训练 20% 的 Q3 Token 效果甚至超过了 100% 全量训练。
*   **Soft-OR 优越性**：结合熵与散度的评分在所有数学推理任务中均优于纯熵筛选。

### 7. 优点
*   **理论支撑**：提供了关于“预测步长 bound”的数学证明，解释了为什么 Q1 > Q2 > Q3 $\gg$ Q4。
*   **极高效率**：在不损失性能的前提下，大幅降低了显存需求，使得在有限 GPU 资源下训练更大模型成为可能。
*   **无参数化**：Soft-OR 公式简单且无需针对不同任务调节超参数。

### 8. 不足与局限
*   **依赖教师分布**：识别 Q3 区域需要获取教师模型的完整输出分布（Logits），这在某些仅提供 API（如 GPT-4）的黑盒蒸馏场景下无法实现。
*   **归一化敏感性**：Soft-OR 依赖 Batch 内的 Min-Max 归一化，可能会受到 Batch 中极端离群 Token 的影响。
*   **损失函数局限**：所有实验均基于 Reverse KL 散度，该分类法在 Forward KL 或其他损失函数下的表现尚待验证。
