## 摘要
Token 剪枝已成为一种有效的方法，通过丢弃信息量较少的视觉 Token 并保持性能，来减少大规模视觉语言模型（LVLMs）巨大的计算开销。然而，现有方法通常依赖于来自 LVLM 不同组件的单一注意力源，由于注意力分布存在偏差，导致剪枝决策不完整且并非最优。为了解决这一问题，我们提出了 DeSAP，一种新型的解耦相似度感知剪枝方法，用于在视觉编码器内进行精确的任务感知 Token 剪枝。具体而言，DeSAP 引入了解耦相似度来捕捉视觉特征与文本 Token 之间细粒度的跨模态相关性，为剪枝提供明确的任务相关引导。通过将解耦相似度与源自视觉注意力的视觉显著性信号相结合，DeSAP 在任务相关线索和视觉线索的共同引导下进行 Token 剪枝，即使在激进的剪枝比例下也能实现鲁棒的剪枝。在不同基准测试和架构上的广泛实验表明，DeSAP 在准确性和效率方面均持续优于现有最先进（SOTA）方法。在 LLaVA-1.5-7B 上，DeSAP 仅保留 11.1% 的视觉 Token，实现了 10 倍的 FLOPs 减少和 2.3 倍的预填充加速，同时保持了 98.1% 的原始性能。

## Abstract
Token pruning has emerged as an effective approach to reduce the substantial computational overhead of Large Vision-Language Models (LVLMs) by discarding less informative visual tokens while preserving performance. However, existing methods typically rely on individual attention sources from different LVLM components, resulting in incomplete and suboptimal pruning decisions due to biased attention distributions. To address this problem, we propose DeSAP, a novel Decoupled Similarity-Aware Pruning method for precise, task-aware token pruning within the visual encoder. Specifically, DeSAP introduces a decoupled similarity to capture fine-grained cross-modal relevance between visual features and text tokens, providing explicit task-related guidance for pruning. By integrating decoupled similarity with visual saliency signals derived from visual attention, DeSAP performs token pruning under the guidance of both task-related and visual cues, enabling robust pruning even under aggressive pruning ratios. Extensive experiments across diverse benchmarks and architectures show that DeSAP consistently outperforms SOTA methods in both accuracy and efficiency. On LLaVA-1.5-7B, DeSAP achieves a 10 times FLOPs reduction and a 2.3 times prefill speedup by retaining only 11.1% of visual tokens, while maintaining 98.1% of the original performance.

### 1. 核心问题与整体含义（研究动机和背景）
随着大规模视觉语言模型（LVLMs）向高分辨率和多帧视频方向发展，视觉 Token 的数量急剧增加，导致了巨大的计算开销和推理延迟。现有的 Token 剪枝方法主要分为两类：
*   **视觉中心（Visual-centric）：** 依赖视觉编码器的注意力（如 `[CLS]` 注意力），容易偏向全局显著语义而忽略任务相关的局部细节。
*   **任务中心（Task-centric）：** 在 LLM 侧利用跨模态注意力，但由于 LLM 的注意力存在空间偏差（倾向于关注序列中邻近的 Token），导致剪枝决策不可靠。
**核心问题：** 如何在视觉编码器端尽早引入准确的任务引导信息，以克服注意力偏差，实现高效且无损的 Token 剪枝？

### 2. 方法论：DeSAP (Decoupled Similarity-Aware Pruning)
DeSAP 的核心思想是**解耦视觉特征**，并利用**细粒度跨模态对齐**来生成任务感知的剪枝引导。
*   **特征解耦：** 将视觉编码器最后一层的隐藏状态分解为“残差组件”（捕获全局语义）和“解耦注意力组件”（保留局部细节）。研究发现，解耦后的注意力组件与文本的相关性更强。
*   **细粒度跨模态对齐：** 
    *   计算解耦后的视觉注意力特征与文本嵌入之间的余弦相似度。
    *   引入 **FCMA（Fine-grained Cross-modal Alignment）** 策略：通过计算 Token 级别的权重贡献，将相似度矩阵进行加权修正，从而更精确地定位与查询相关的区域。
*   **双源引导剪枝：** 
    *   将上述得到的“任务相关相似度图”与传统的“`[CLS]` 全局显著性图”结合。
    *   按比例（默认各 50%）分配剪枝预算，同时保留任务关键区域和全局背景上下文。
*   **Token 合并：** 对于未被选中的 Token，采用基于聚类的合并策略，将其信息聚合到最相似的保留 Token 中，以减少信息损失。

### 6. 主要结论与发现
*   **性能卓越：** 在 LLaVA-1.5-7B 上，仅保留 11.1% 的 Token（64个），平均准确率仍能达到原始模型的 98.1%；在保留 192 个 Token 时，性能甚至超过了全量模型。
*   **效率提升：** 实现了 10 倍的 FLOPs 减少，预填充阶段提速 2.3 倍，整体推理提速 1.6 倍。
*   **鲁棒性：** 相比于其他方法，DeSAP 在处理小物体和复杂任务相关查询时表现更稳健，能够更精准地保留任务关键信息。

### 7. 优点（亮点）
*   **创新的解耦视角：** 首次提出在编码器端通过解耦隐藏状态来提取更纯净的任务引导信号。
*   **双源互补：** 结合了任务感知和视觉显著性，解决了单一注意力源带来的偏差问题。
*   **通用性强：** 无需重新训练，可直接应用于多种主流 LVLM 架构（如 LLaVA 系列和 Qwen 系列）。
*   **细粒度对齐：** FCMA 策略有效提升了跨模态匹配的精度。

### 8. 不足与局限
*   **对文本查询的依赖：** 当文本查询非常模糊（如简单的“描述这张图”）时，任务引导信号会变弱，模型性能会更多地退化到依赖 `[CLS]` 注意力。
*   **长文本开销：** 虽然引入了文本 Token 筛选策略，但极长的文本输入仍会增加相似度计算的负担。
*   **动态比例：** 目前任务引导和视觉引导的比例是固定分配的（50/50），未来若能根据任务类型动态调整比例可能会进一步提升效果。
