## 摘要
视觉语言模型（VLM）通过联合处理视觉和文本信息，彻底改变了多模态学习。然而，由于处理长序列视觉 Token 对计算和内存的高需求，它们面临着重大挑战。许多现有方法依赖于局部启发式方法，例如注意力分数或 Token 范数。然而，这些标准存在位置偏差和信息分散的问题，限制了它们在高剪枝率下保留核心内容的能力，并导致在视觉细节丰富的图像上性能下降。为了解决这些问题，我们提出了 SVD-Prune，一种基于奇异值分解（SVD）的免训练、即插即用的 Token 剪枝方法。该方法通过分解视觉 Token 特征矩阵，并利用统计杠杆得分（statistical leverage scores）选择前 K 个 Token，确保仅保留对主要全局方差贡献最大的 Token。实验表明，在极端的视觉 Token 预算下，SVD-Prune 始终优于先前的剪枝方法，即使在仅剩 32 和 16 个视觉 Token 的情况下也能保持强大的性能。

## Abstract
Vision-Language Models (VLM) have revolutionized multimodal learning by jointly processing visual and textual information. Yet, they face significant challenges due to the high computational and memory demands of processing long sequences of vision tokens. Many existing methods rely on local heuristics, such as attention scores or token norms. However, these criteria suffer from positional bias and information dispersion, limiting their ability to preserve essential content at high pruning ratios and leading to performance degradation on visually detailed images. To address these issues, we propose SVD-Prune, a trainingfree, plug-and-play token pruning method based on Singular Value Decomposition. It decomposes the vision token feature matrix and selects the top-K tokens using statistical leverage scores, ensuring only tokens contributing most to the dominant global variance are preserved. Experiments show that SVD-Prune consistently outperforms prior pruning methods under extreme vision token budgets, maintaining strong performance even with 32 and 16 vision tokens.

*   **背景**：视觉语言模型（VLM）在处理图像时会生成大量视觉 Token（如 LLaVA-1.5 默认生成 576 个），这导致了巨大的计算开销和内存占用，尤其是在资源受限的边缘设备上。
*   **核心问题**：现有的 Token 剪枝方法（如基于注意力分数、Token 范数或相似度的方法）存在两大缺陷：
    1.  **位置偏差（Positional Bias）**：由于 LLM 解码器的因果掩码（Causal Masking），靠后的 Token 往往获得较低的注意力分数，导致剪枝决策受位置而非语义驱动。

*   **目标**：开发一种无需训练、即插即用、且能克服位置偏差的高效剪枝方案。

### 2. 方法论：SVD-Prune 核心思想与流程
该方法的核心是利用**奇异值分解（SVD）**从全局角度识别最具代表性的视觉 Token。其流程分为四个阶段：
1.  **全局模式提取**：将视觉编码器输出的特征矩阵 $F$ 进行 SVD 分解，得到左奇异向量 $U$、奇异值矩阵 $\Sigma$ 和右奇异向量 $V^\top$。这能捕捉 Token 空间中的主成分方向。
2.  **主方差截断**：计算累积解释方差比例，选择最小的秩 $k$，使得前 $k$ 个奇异值保留了原始特征矩阵中大部分的方差（超参数 $\epsilon$ 控制，通常在 0.7 到 0.95 之间）。
3.  **杠杆得分计算（Leverage Scores）**：利用前 $k$ 个左奇异向量计算每个 Token 的统计杠杆得分 $\ell_t$。得分越高，代表该 Token 对全局主成分的贡献越大，即信息量越丰富。
4.  **Token 选择与重排**：根据得分降序排列并选择累积得分达到阈值的最小 Token 集合。最后，将保留的 Token **按原始空间顺序重排**，以保持位置编码的一致性。
