## 摘要
视觉自回归（VAR）建模在生成高分辨率图像时，对每个位置应用固定的计算深度，导致效率低下。虽然现有方法通过频率图剪枝 token 来加速推理，但其二值化的硬剪枝（hard-pruning）方法存在根本局限，即使改进频率估计也难以提升质量。基于 VAR 模型存在显著深度冗余的观察，我们提出了一种范式转变：从剪枝整个 token 转向自适应分配每个 token 的计算深度。为此，我们引入了 DepthVAR，这是一个无需训练的动态计算分配框架。它集成了一个自适应深度调度器，通过循环旋转调度分配计算深度以实现平衡且非静态的细化，并结合动态推理过程将深度转换为层优先掩码，选择性地应用 Transformer 块并融合生成的代码，确保每个 token 的影响力与其处理深度成正比。大量实验表明，DepthVAR 在质量损失极小的情况下实现了 2.3-3.1 倍的加速，与现有的硬剪枝方法相比，提供了更具竞争力的计算-性能权衡。代码开源于 https://github.com/STOVAGtz/DepthVAR。

## Abstract
Visual Autoregressive (VAR) modeling inefficiently applies a fixed computational depth to each position when generating high-resolution images. While existing methods accelerate inference by pruning tokens using frequency maps, their binary hard-pruning approach is fundamentally limited and fails to improve quality even with better frequency estimation. Observing that VAR models possess significant depth redundancy, we propose a paradigm shift from pruning entire tokens to adaptively allocating per-token computational depth. To this end, we introduce DepthVAR, a training-free framework that dynamically allocates computation. It integrates an adaptive depth scheduler, which assigns computational depth via a cyclic rotated schedule for balanced, non-static refinement, with a dynamic inference process that translates these depths into layer-major masks, selectively applies transformer blocks, and blends the resulting codes to ensure each token's influence is proportional to its processing depth. Extensive experiments show that DepthVAR achieves 2.3$\times$-3.1$\times$ acceleration with minimal quality loss, offering a competitive compute-performance trade-off compared to existing hard-pruning approaches. Code is available at https://github.com/STOVAGtz/DepthVAR

#### 1. 论文的核心问题与整体含义
视觉自回归（VAR）模型通过“逐尺度预测”生成图像，虽然比传统的逐 token 预测更高效，但在处理高分辨率图像时，对所有空间位置（token）应用相同的计算深度（即通过所有 Transformer 层）会导致严重的计算冗余。
*   **研究动机**：现有的加速方案（如 FastVAR、SparseVAR）主要采用“硬剪枝”策略，即根据频率图简单地保留或丢弃 token。作者发现，这种二值化的处理方式存在根本局限：即使拥有完美的频率估计，硬剪枝仍会导致明显的图像质量下降。
*   **核心问题**：如何在不牺牲图像质量的前提下，更精细地分配计算资源？作者观察到 VAR 模型在深度上存在显著冗余，因此提出将“token 剪枝”范式转变为“自适应深度分配”范式。

#### 2. 论文提出的方法论
作者提出了 **DepthVAR**，一个无需训练（Training-free）的推理加速框架，其核心流程如下：

*   **自适应深度分数调度器 (Adaptive Depth Score Scheduler)**：
    *   利用前一尺度的层间特征变化作为“决策参考”，计算每个位置的决策秩（Rank）。
    *   引入**循环旋转调度 (Cyclic Rotated Schedule)**：通过对百分位秩进行循环旋转，确保图像各区域能得到平衡且持续的细化，避免某些区域被永久性地分配低计算量。
*   **位反转层优先掩码 (Bit-reversal Layer-major Mask)**：
    *   将连续的深度分数转换为整数层数，并利用**位反转置换（Bit-reversal Permutation）**算法将活跃层均匀分布在整个网络中。这种设计确保了层利用的无偏性，防止浅层 token 集中在网络的特定部分。
*   **动态推理与代码融合 (Dynamic Inference & Code Blending)**：
    *   **掩码推理**：仅在掩码指定的活跃层处理 token，非活跃层则复用前一尺度的缓存行为。
    *   **深度加权融合**：在输出端，根据每个 token 的处理深度对其生成的代码（Code）进行加权融合。处理深度越深，对最终残差的影响力越大，从而实现连续的计算缩放。

#### 6. 论文的主要结论与发现
*   **深度冗余普遍存在**：实验证明 VAR 模型在生成后期存在严重的深度冗余，图像质量往往在到达最后一层前就已达到峰值。
*   **连续分配优于硬剪枝**：通过为每个 token 分配不同的计算深度，比直接丢弃 token 能更好地保留图像的结构细节。
*   **加速效果显著**：在 Infinity 模型上实现了 2.3x-3.1x 的加速，且在 GenEval 等语义指标上几乎没有损失，甚至在某些配置下有所提升。

#### 7. 优点（亮点）
*   **范式创新**：打破了传统的“保留/丢弃”二值化剪枝思维，引入了连续的深度分配机制。
*   **无需训练**：作为一种即插即用的推理插件，极易集成到现有的预训练 VAR 模型中。
*   **调度巧妙**：利用位反转算法解决层分配的均匀性问题，利用循环旋转解决空间更新的平衡性问题，逻辑严密。

#### 8. 不足与局限
*   **固定总预算**：目前的计算预算是按样本预设的（通过参考尺度 R 控制），无法根据图像内容的实际复杂度动态调整总计算量（例如简单背景应自动减少总预算）。
*   **精细细节损失**：在极高加速比下，对于极其密集的结构或微小细节（如远景中的小物体），仍可能出现细微的质量下降。
*   **缓存开销**：为了保持层行为的连续性，需要缓存前一尺度的特征，这增加了一定的显存负担。
