## 摘要
减少序列长度是加速 Transformer 的常用方法，但以往的令牌（token）缩减工作通常针对分类任务，且报告的是代理指标而非端到端延迟。对于语义分割，令牌缩减受到重建密集、像素对齐特征需求的进一步约束，且在现代加速器上，计算合并映射（merge maps）的开销可能会抵消预期的收益。我们提出了互对合并（Mutual Pair Merging, MPM），这是一种无需训练的令牌聚合模块。它在余弦空间中形成互近邻对，对每对进行平均，并记录合并映射，从而在解码器之前实现基于 gather 操作的重建，使得现有的分割头可以无需修改直接使用。MPM 不引入学习参数，也没有连续的压缩调节旋钮（如保留率或阈值）。速度与精度的权衡由离散的插入调度（insertion schedule）决定。我们在 NVIDIA H100 GPU（包含和不包含 FlashAttention-2）以及 Raspberry Pi 5 上，针对标准分割数据集基准测试了端到端延迟。在 ADE20K 数据集上，对于 Raspberry Pi 5 上的 ViT-Tiny，MPM 将单张图像延迟降低了高达 60%；在带有 FlashAttention-2 的 H100 上，吞吐量提升了高达 20%，同时将 mIoU 的下降控制在 3% 以内。这些结果表明，在明确考虑开销的情况下，简单的、感知重建的、无需训练的令牌合并可以转化为分割任务中实际的耗时收益。

## Abstract
Decreasing sequence length is a common way to accelerate transformers, but prior token reduction work often targets classification and reports proxy metrics rather than end-to-end latency. For semantic segmentation, token reduction is further constrained by the need to reconstruct dense, pixel-aligned features, and on modern accelerators the overhead of computing merge maps can erase expected gains. We propose Mutual Pair Merging (MPM), a training-free token aggregation module that forms mutual nearest-neighbor pairs in cosine space, averages each pair, and records a merge map enabling a gather-based reconstruction before the decoder so that existing segmentation heads can be used unchanged. MPM introduces no learned parameters and no continuous compression knob (no keep-rate or threshold). The speed-accuracy trade-off is set by a discrete insertion schedule. We benchmark end-to-end latency on an NVIDIA H100 GPU (with and without FlashAttention-2) and a Raspberry Pi 5 across standard segmentation datasets. On ADE20K, MPM reduces per-image latency by up to 60% for ViT-Tiny on Raspberry Pi 5, and increases throughput by up to 20% on H100 with FlashAttention-2 while keeping the mIoU drop below 3%. These results suggest that simple, reconstruction-aware, training-free token merging can translate into practical wall-clock gains for segmentation when overhead is explicitly accounted for.

    2.  **实际开销**：在现代 GPU（如 H100）上，计算合并映射（Merge Maps）的额外开销往往会抵消 Token 减少带来的加速效果。

### 2. 方法论：核心思想与关键技术
MPM 是一种即插即用的模块，其核心流程如下：
*   **互近邻对合并（Mutual Pair Merging）**：
    1.  **余弦相似度计算**：对输入 Token 进行 L2 归一化，计算全量余弦相似度矩阵。
    2.  **互近邻匹配**：对于 Token $i$，寻找其最相似的邻居 $b(i)$。如果 $b(b(i)) = i$，则这两个 Token 形成一个“互近邻对”。
    3.  **代表元平均**：将互近邻对中的两个 Token 取平均值合并为一个新 Token，未形成互对的 Token 作为单例（Singleton）保留。
*   **确定性与免参数**：该过程不含学习参数，不设连续的压缩阈值，完全由数据驱动且结果确定。
*   **合并映射与重建**：
    *   MPM 会记录一个整数索引映射表（Merge Map）。
    *   在编码器结束、进入解码器之前，通过一个简单的 **Gather（聚合）操作**，根据映射表将合并后的特征“拷贝回”原始的 $H/P \times W/P$ 网格位置。
    *   这种设计使得 MPM 可以直接适配现有的分割头（如 Mask Transformer），无需修改架构。
*   **插入策略**：通过在 Transformer 的特定层（默认第 3 层和第 6 层）插入 MPM 来控制速度与精度的权衡。

*   **端到端导向**：不玩弄 FLOPs 数字游戏，直接报告真实硬件上的 FPS，对工业界部署非常友好。

### 8. 不足与局限
*   **边界平滑风险**：过度合并可能导致细小物体或物体边界的预测变得模糊。
*   **计算复杂度**：计算全量余弦相似度矩阵仍具有 $O(N^2)$ 复杂度，虽然在 16x16 补丁下开销尚可，但在极高分辨率（Token 极多）时可能成为瓶颈。
*   **精度权衡**：相比于需要微调的方法（如 ALGM），免训练的 MPM 在相同加速比下精度略低。
*   **架构限制**：仅适用于全局注意力的 Plain ViT，对于 Swin 等具有线性复杂度的层次化架构，Token 缩减的收益微乎其微。
