## 摘要
Token 剪枝已成为降低大语言视觉模型（LVLMs）巨大计算开销的一种有效方法，它通过丢弃信息量较少的视觉 Token，同时保持模型性能。然而，现有方法通常依赖于来自 LVLM 不同组件的单一注意力源，由于注意力分布存在偏差，导致剪枝决策不完整且并非最优。为解决这一问题，我们提出了 DeSAP，一种新颖的解耦相似度感知剪枝方法，用于在视觉编码器内进行精确的任务感知 Token 剪枝。具体而言，DeSAP 引入了解耦相似度来捕捉视觉特征与文本 Token 之间细粒度的跨模态相关性，为剪枝提供明确的任务相关引导。通过将解耦相似度与源自视觉注意力的视觉显著性信号相结合，DeSAP 在任务相关线索和视觉线索的双重引导下进行 Token 剪枝，即使在激进的剪枝比例下也能实现稳健的剪枝。在不同基准测试和架构上的广泛实验表明，DeSAP 在准确率和效率方面均持续优于 SOTA 方法。在 LLaVA-1.5-7B 上，DeSAP 仅保留 11.1% 的视觉 Token，实现了 10 倍的 FLOPs 减少和 2.3 倍的预填充加速，同时保持了 98.1% 的原始性能。

## Abstract
Token pruning has emerged as an effective approach to reduce the substantial computational overhead of Large Vision-Language Models (LVLMs) by discarding less informative visual tokens while preserving performance. However, existing methods typically rely on individual attention sources from different LVLM components, resulting in incomplete and suboptimal pruning decisions due to biased attention distributions. To address this problem, we propose DeSAP, a novel Decoupled Similarity-Aware Pruning method for precise, task-aware token pruning within the visual encoder. Specifically, DeSAP introduces a decoupled similarity to capture fine-grained cross-modal relevance between visual features and text tokens, providing explicit task-related guidance for pruning. By integrating decoupled similarity with visual saliency signals derived from visual attention, DeSAP performs token pruning under the guidance of both task-related and visual cues, enabling robust pruning even under aggressive pruning ratios. Extensive experiments across diverse benchmarks and architectures show that DeSAP consistently outperforms SOTA methods in both accuracy and efficiency. On LLaVA-1.5-7B, DeSAP achieves a 10 times FLOPs reduction and a 2.3 times prefill speedup by retaining only 11.1% of visual tokens, while maintaining 98.1% of the original performance.

*   **现有方法的局限性**：
    *   **视觉中心（Visual-centric）方法**：依赖视觉编码器的 `[CLS]` 注意力，容易偏向全局语义，忽略任务相关的局部细节。
    *   **任务中心（Task-centric）方法**：在 LLM 侧利用跨模态注意力，但 LLM 早期层的注意力存在严重的“空间偏置”（倾向于关注序列末尾/图像底部的 Token），导致关键信息过早丢失。
*   **核心动机**：探索如何在视觉编码器端（Encoder-side）尽早引入任务引导信息，以实现更精确、无偏的 Token 剪枝。

### 2. 方法论：DeSAP (Decoupled Similarity-Aware Pruning)
DeSAP 的核心思想是**解耦视觉特征**，并利用**细粒度的跨模态对齐**来引导剪枝。

*   **特征解耦（Decomposition）**：将视觉编码器最后一层的隐藏状态分解为“解耦残差组件”（捕获全局语义）和“解耦注意力组件”（保留局部细节）。研究发现，去除残差连接后的注意力特征能更好地与文本语义对齐。
*   **细粒度跨模态对齐（FCMA）**：
    *   计算解耦后的视觉特征与文本嵌入之间的余弦相似度。
    *   引入 Token 级别的权重贡献机制（受 TokenFlow 启发），通过衡量每个 Token 与对方模态全局代表性特征（如视觉的 `[CLS]` 和文本的 `[EOS]`）的相关性，来重新加权相似度矩阵，从而获得更精确的“解耦相似度”。
*   **双源引导剪枝（Dual-source Guidance）**：
    *   结合**任务相关相似度图**（Decoupled Similarity）和**视觉显著性图**（[CLS] Attention）。
    *   采用等比例分配预算的策略，分别从两个图中筛选出最重要的 Token，并对未选中的 Token 进行基于聚类的均值合并（Token Merging），以减少信息损失。

### 6. 主要结论与发现
*   **解耦相似度更可靠**：视觉编码器的隐藏状态受全局语义偏置严重，而解耦后的注意力特征能提供更精准的任务引导。
*   **性能卓越**：在 LLaVA-1.5-7B 上，即使剪枝掉 88.9% 的 Token（仅保留 64 个），仍能保持 98.1% 的性能；在保留 192 个 Token 时，性能甚至超过了全量 Token 模型（100.3%）。
*   **通用性强**：在不同分辨率（LLaVA-NeXT）和不同架构（Qwen-2.5-VL）下均表现出领先的 SOTA 性能。

### 7. 优点（亮点）
*   **创新视角**：首次提出在视觉编码器端通过解耦特征来实现细粒度的跨模态对齐，解决了 LLM 侧剪枝存在的空间偏置问题。
*   **高效且免训练**：无需任何微调即可直接应用于现有模型，计算开销极低（相比 backbone 计算量可忽略不计）。
*   **鲁棒性**：通过双源引导（任务+视觉），在极端剪枝比例下依然能保留关键的局部细节（如图像中的微小物体）。

### 8. 不足与局限
*   **文本依赖性**：当用户查询（Query）语义模糊或过于简短（如 Caption 任务）时，任务引导的效果会减弱，模型会退化为依赖视觉显著性。
*   **长文本开销**：虽然引入了文本 Token 筛选策略，但处理极长文本输入时，跨模态相似度计算的复杂度仍会随文本长度增加。
*   **超参数分配**：目前任务相关和视觉显著性 Token 的分配比例是固定的（50/50），未来可能需要更动态的分配机制。
