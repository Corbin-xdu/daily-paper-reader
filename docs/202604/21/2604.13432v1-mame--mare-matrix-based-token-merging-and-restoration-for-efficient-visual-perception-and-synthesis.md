## 摘要
Token 压缩对于缓解视觉 Transformer (ViT) 中自注意力机制的二次复杂度至关重要，因为 ViT 通常涉及大量的输入 Token。现有方法（如 ToMe）依赖于 GPU 效率较低的操作（例如排序、离散写入），从而引入了限制其有效性的额外开销。我们提出了 MaMe，这是一种完全基于矩阵操作的、无需训练且可微的 Token 合并方法，它对 GPU 友好，可用于加速 ViT。此外，我们提出了其逆操作 MaRe 用于 Token 恢复，从而构建了一个用于图像合成的 MaMe+MaRe 流水线。当应用于预训练模型时，MaMe 在准确率仅下降 2% 的情况下使 ViT-B 的吞吐量翻倍。值得注意的是，使用 MaMe 对最后一层进行微调，在 1.1 倍速度下将 ViT-B 的准确率提升了 1.0%。在 SigLIP2-B@512 零样本分类中，MaMe 提供了 1.3 倍的加速，且性能下降可以忽略不计。在视频任务中，MaMe 在 Kinetics-400 数据集上将 VideoMAE-L 加速了 48.5%，而准确率损失仅为 0.84%。此外，MaMe 在某些任务上实现了性能和速度的双重提升。在图像合成中，MaMe+MaRe 流水线在提高质量的同时，将 Stable Diffusion v2.1 的生成延迟降低了 31%。总之，这些结果证明了 MaMe 和 MaRe 在加速视觉模型方面的有效性。代码可在 https://github.com/cominder/mame 获取。

## Abstract
Token compression is crucial for mitigating the quadratic complexity of self-attention mechanisms in Vision Transformers (ViTs), which often involve numerous input tokens. Existing methods, such as ToMe, rely on GPU-inefficient operations (e.g., sorting, scattered writes), introducing overheads that limit their effectiveness. We introduce MaMe, a training-free, differentiable token merging method based entirely on matrix operations, which is GPU-friendly to accelerate ViTs. Additionally, we present MaRe, its inverse operation, for token restoration, forming a MaMe+MaRe pipeline for image synthesis. When applied to pre-trained models, MaMe doubles ViT-B throughput with a 2% accuracy drop. Notably, fine-tuning the last layer with MaMe boosts ViT-B accuracy by 1.0% at 1.1x speed. In SigLIP2-B@512 zero-shot classification, MaMe provides 1.3x acceleration with negligible performance degradation. In video tasks, MaMe accelerates VideoMAE-L by 48.5% on Kinetics-400 with only a 0.84% accuracy loss. Furthermore, MaMe achieves simultaneous improvements in both performance and speed on some tasks. In image synthesis, the MaMe+MaRe pipeline enhances quality while reducing Stable Diffusion v2.1 generation latency by 31%. Collectively, these results demonstrate MaMe's and MaRe's effectiveness in accelerating vision models. The code is available at https://github.com/cominder/mame}{https://github.com/cominder/mame.

Vision Transformers (ViTs) 在处理高分辨率图像或长视频时，其自注意力机制的计算复杂度随 Token 数量 $N$ 呈二次方增长（$O(N^2)$）。现有的 Token 压缩技术（如 ToMe）虽然能减少 Token 数量，但往往依赖于**排序（Sorting）**、**散列写入（Scattered Writes）**或**聚类（Clustering）**等操作。这些操作在 GPU 上执行效率较低，导致理论上的计算量减少无法转化为实际的推理加速。
**研究动机：** 开发一种完全基于**矩阵运算**、**对 GPU 友好**、**可微**且**无需训练**的 Token 合并（MaMe）与恢复（MaRe）方法，以实现真正的端到端推理加速。

### 2. 方法论：核心思想与关键技术
论文提出了两个互补的操作：**MaMe**（合并）和 **MaRe**（恢复）。

*   **MaMe (Matrix-Based Token Merging):**
    1.  **Token 分区：** 将输入 Token 划分为目标集合（Destination）和源集合（Source）。
    2.  **相似度计算：** 计算两组 Token 之间的余弦相似度矩阵，并通过 ReLU 和阈值 $\tau$ 进行稀疏化。
    3.  **自适应权重精炼：** 通过列归一化和动态阈值处理，生成融合权重矩阵 $W_F$。
    4.  **Token 聚合与保留：** 根据权重将源 Token 融合进目标 Token。特别地，如果某个源 Token 与所有目标 Token 都不相似，则将其作为“独特信息”原样保留，避免信息丢失。
*   **MaRe (Matrix-Based Token Restoration):**
    *   利用 MaMe 存储的融合矩阵 $W_F$，通过逆向加权分配，将合并后的 Token 重新映射回原始的 Token 空间。这对于需要保持空间分辨率的任务（如图像生成）至关重要。
*   **集成方式：** 在感知任务中仅使用 MaMe；在生成任务（如 Stable Diffusion）中，在注意力层前使用 MaMe 压缩，层后使用 MaRe 恢复。
