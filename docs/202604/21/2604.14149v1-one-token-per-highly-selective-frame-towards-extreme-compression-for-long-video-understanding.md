## 摘要
由于视频帧数庞大，长视频理解对视觉语言模型（VLM）而言具有内在挑战性。由于每个视频帧通常会扩展为数十或数百个 token，大语言模型（LLM）有限的上下文长度迫使 VLM 只能稀疏地感知视频帧，从而丢失了时序信息。为解决这一问题，我们探索了在 LLM 最终层实现“每帧一个 token”的极致视频 token 压缩。我们的核心见解是，先前方法广泛采用的基于启发式的压缩容易导致信息丢失，因此有必要将 LLM 层监督为用于“token 级压缩”的“可学习”且“渐进式”模块（LP-Comp）。这种压缩使我们的 VLM 能够处理 2 到 4 倍的视频帧，并提升性能。为了进一步提高 token 效率，我们研究了“帧级压缩”，即通过 LLM 层的内部注意力分数选择与查询最相关的帧，称之为“问题条件压缩”（QC-Comp）。与以往研究的一个显著区别在于，我们通过将长视频分割成短片段并采用局部注意力，缓解了长上下文中 LLM 注意力的位置偏差（即过度集中在序列开头和结尾的问题）。总之，我们结合了 token 级和帧级压缩，构建了一个名为 X-Comp 的长视频理解极致压缩模型，实现了显著更高的压缩率并支持更密集的帧采样。我们的 X-Comp 基于 VideoChat-Flash 进行微调，采用了一种数据高效的“监督压缩微调”阶段，仅需 2.5% 的监督微调数据，便将 LVBench 上的准确率从 42.9% 提升至 46.2%，并增强了多个其他长视频基准测试的表现。

## Abstract
Long video understanding is inherently challenging for vision-language models (VLMs) because of the extensive number of frames. With each video frame typically expanding into tens or hundreds of tokens, the limited context length of large language models (LLMs) forces the VLMs to perceive the frames sparsely and lose temporal information. To address this, we explore extreme video token compression towards \emph{one token per frame} at the final LLM layer. Our key insight is that heuristic-based compression, widely adopted by previous methods, is prone to information loss, and this necessitates supervising LLM layers into \emph{learnable} and \emph{progressive} modules for \emph{token-level compression} (LP-Comp). Such compression enables our VLM to digest 2x-4x more frames with improved performance. To further increase the token efficiency, we investigate \emph{frame-level compression}, which selects the frames most relevant to the queries via the internal attention scores of the LLM layers, named \emph{question-conditioned compression} (QC-Comp). As a notable distinction from previous studies, we mitigate the position bias of LLM attention in long contexts, \emph{i.e.}, the over-concentration on the beginning and end of a sequence, by splitting long videos into short segments and employing local attention. Collectively, our combined \emph{token-level} and \emph{frame-level} leads to an e\textbf{x}treme compression model for long video understanding, named \textbf{\name}, achieving a significantly larger compression ratio and enabling denser frame sampling. Our \name is finetuned from VideoChat-Flash with a data-efficient \emph{supervised compression tuning} stage that only requires 2.5\% of the supervised fine-tuning data, yet boosts the accuracy from 42.9\% to 46.2\% on LVBench and enhances multiple other long video benchmarks.

### 1. 核心问题与整体含义（研究动机和背景）
长视频理解的主要瓶颈在于**视觉 Token 爆炸**。现有的视觉语言模型（VLM）在处理每一帧时通常会生成数十甚至数百个 Token，这迅速填满了大语言模型（LLM）有限的上下文窗口。为了适应长视频，现有方法通常不得不采取极稀疏的采样（如每秒不到一帧），导致关键时序信息丢失。
*   **核心挑战**：如何在不损失关键信息的前提下，将视频压缩到极致，从而允许模型输入更密集的帧？
*   **研究动机**：作者认为传统的启发式压缩（如简单的池化或基于规则的丢弃）会导致不可逆的信息损失。因此，需要让 LLM 自身学会如何将一帧的信息“浓缩”到一个 Token 中。

### 2. 方法论：核心思想与关键技术
论文提出了名为 **XComp** 的框架，从 Token 级和帧级两个维度进行极致压缩：

*   **LP-Comp（可学习且渐进式 Token 压缩）**：
    *   **核心思想**：不再依赖预定义的规则，而是通过监督微调让 LLM 的每一层都参与压缩。
    *   **技术细节**：采用**后缀保留（Suffix-Preservation）**策略。由于 Decoder-only LLM 具有因果注意力机制，后面的 Token 可以吸收前面 Token 的信息。模型在每一层根据**余弦调度曲线（Cosine Schedule）**逐渐减少每帧的 Token 数，直到最后一层每帧仅剩 1 个 Token。
    *   **监督压缩微调（SCT）**：通过端到端训练，使 LLM 层具备将多 Token 信息压缩进少数保留 Token 的能力。

*   **QC-Comp（问题引导的帧级压缩）**：
    *   **核心思想**：利用 LLM 内部的注意力分数来识别哪些帧对回答当前问题最重要。
    *   **分段局部注意力（Segmented Local Attention）**：为解决长文本中常见的“迷失在中间”（Lost-in-the-middle）位置偏差问题，将长视频切分为短片段，在片段内计算问题 Token 与视频 Token 的注意力权重，从而更公平地筛选全局关键帧。

### 6. 主要结论与发现
*   **极致压缩可行性**：通过学习，LLM 完全可以将一帧的信息压缩至 1 个 Token 而不显著损失理解能力。
*   **性能提升**：XComp 在 LVBench 上将准确率从 42.9% 提升至 46.2%，在 2B 规模模型中达到 SOTA 水平。
*   **正向反馈循环**：由于 Token 效率极高，模型可以采样更密集的帧（如 2x-4x 更多），而更密集的帧进一步提升了长视频理解的精度，打破了以往“帧数增加导致性能下降”的怪圈。

### 7. 优点（亮点）
*   **数据高效性**：仅需极少量的微调数据即可赋予模型强大的压缩能力。
*   **渐进式设计**：LP-Comp 的余弦调度比突发式压缩更平滑，有效保留了特征。
*   **解决位置偏差**：QC-Comp 通过局部注意力机制巧妙地解决了 LLM 在长序列中对中间信息关注不足的问题。
*   **端到端一致性**：压缩过程完全集成在 LLM 内部，不需要额外的复杂插件。

### 8. 不足与局限
*   **模型规模限制**：主要实验集中在 2B 规模，虽然在 LLaVA-Next 上做了验证，但尚未在大规模（如 70B+）模型上进行大规模预训练验证。
*   **任务偏差**：在某些需要极精细局部特征的任务（如 VideoOCR）中，极致压缩可能会带来轻微的性能下降。
*   **推理开销**：虽然减少了 LLM 的计算量，但 QC-Comp 需要先进行一次前向传递来计算注意力分数，这在实时性要求极高的场景下可能存在额外开销。
