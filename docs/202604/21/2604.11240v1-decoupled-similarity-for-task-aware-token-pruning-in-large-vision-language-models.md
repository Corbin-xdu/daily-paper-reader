## 摘要
Token 剪枝已成为一种有效的方法，通过丢弃信息量较少的视觉 Token 并保持性能，来减少大语言视觉模型（LVLMs）巨大的计算开销。然而，现有方法通常依赖于来自不同 LVLM 组件的单一注意力源，由于注意力分布存在偏差，导致剪枝决策不完整且并非最优。为了解决这一问题，我们提出了 DeSAP，一种新型的解耦相似度感知剪枝方法，用于在视觉编码器内进行精确的任务感知 Token 剪枝。具体而言，DeSAP 引入了解耦相似度来捕获视觉特征与文本 Token 之间细粒度的跨模态相关性，为剪枝提供明确的任务相关引导。通过将解耦相似度与源自视觉注意力的视觉显著性信号相结合，DeSAP 在任务相关线索和视觉线索的共同引导下进行 Token 剪枝，即使在激进的剪枝比例下也能实现鲁棒的剪枝。在不同基准测试和架构上的广泛实验表明，DeSAP 在准确性和效率方面均持续优于现有最先进（SOTA）方法。在 LLaVA-1.5-7B 上，DeSAP 仅保留 11.1% 的视觉 Token，就实现了 10 倍的 FLOPs 减少和 2.3 倍的预填充加速，同时保持了 98.1% 的原始性能。

## Abstract
Token pruning has emerged as an effective approach to reduce the substantial computational overhead of Large Vision-Language Models (LVLMs) by discarding less informative visual tokens while preserving performance. However, existing methods typically rely on individual attention sources from different LVLM components, resulting in incomplete and suboptimal pruning decisions due to biased attention distributions. To address this problem, we propose DeSAP, a novel Decoupled Similarity-Aware Pruning method for precise, task-aware token pruning within the visual encoder. Specifically, DeSAP introduces a decoupled similarity to capture fine-grained cross-modal relevance between visual features and text tokens, providing explicit task-related guidance for pruning. By integrating decoupled similarity with visual saliency signals derived from visual attention, DeSAP performs token pruning under the guidance of both task-related and visual cues, enabling robust pruning even under aggressive pruning ratios. Extensive experiments across diverse benchmarks and architectures show that DeSAP consistently outperforms SOTA methods in both accuracy and efficiency. On LLaVA-1.5-7B, DeSAP achieves a 10 times FLOPs reduction and a 2.3 times prefill speedup by retaining only 11.1% of visual tokens, while maintaining 98.1% of the original performance.

### 1. 核心问题与整体含义（研究动机和背景）
大语言视觉模型（LVLMs）在处理高分辨率图像或多帧视频时，会产生海量的视觉 Token，导致巨大的计算开销和推理延迟。
*   **现有挑战**：目前的 Token 剪枝方法主要分为“视觉中心”（依赖视觉编码器的 [CLS] 注意力）和“任务中心”（依赖 LLM 的跨模态注意力）。
*   **研究动机**：作者发现现有方法存在**注意力偏差**：视觉编码器的注意力往往偏向全局语义而忽略局部细节；而 LLM 的跨模态注意力在早期层存在严重的“空间偏见”（倾向于关注图像底部靠近文本的 Token）。
*   **核心目标**：在视觉编码器端提前引入“任务感知”引导，通过消除注意力偏差，实现更精准、高效的 Token 剪枝。

### 2. 方法论：DeSAP
DeSAP（Decoupled Similarity-Aware Pruning）的核心思想是将视觉特征解耦，并结合任务引导进行双源剪枝。
*   **隐藏状态解耦**：将视觉编码器最后一层的隐藏状态分解为“残差分量”（捕获全局语义）和“注意力分量”（保留局部细节）。实验证明，注意力分量能更好地与文本进行细粒度对齐。
*   **解耦相似度计算**：利用解耦后的注意力分量与文本嵌入计算余弦相似度。为了增强鲁棒性，引入了**细粒度跨模态对齐（FCMA）策略**，通过计算 Token 级别的加权贡献来生成任务相关的注意力图。
*   **双源引导剪枝**：
    1.  **任务感知分支**：根据解耦相似度选择与查询最相关的 Token。
    2.  **视觉显著性分支**：根据 [CLS] 注意力选择图像中视觉上最显著的 Token。
    3.  **融合策略**：两者各占 50% 的配额，确保模型既能看到任务目标，也能保留必要的背景上下文。
*   **Token 合并**：未被选中的 Token 不会被直接丢弃，而是通过聚类方式合并到最相似的保留 Token 中，以最大程度减少信息损失。

### 3. 实验设计
*   **数据集/场景**：涵盖了 10 个标准基准测试，包括图像理解（GQA, VQAv2, TextVQA, VizWiz）、多模态推理（MMBench, MME, POPE, ScienceQA）以及视频问答（MSVD-QA, MSRVTT-QA）。
*   **模型架构（Backbones）**：验证了 LLaVA-1.5-7B、LLaVA-NeXT-7B、Video-LLaVA-7B 以及最新的 Qwen-2.5-VL-7B。
*   **对比方法（Baselines）**：对比了 9 种 SOTA 方法，包括视觉中心的 VisionZip、FlowCut、HoloV、LLaVA-PruMerge，以及任务中心的 SparseVLM、PyramidDrop、V²Drop 等。

*   **硬件环境**：所有实验均在 **Nvidia H20-96G GPU** 上进行。
*   **软件环境**：Python 3.10.18, PyTorch 2.1.2, CUDA 12.1。
*   **训练时长**：该方法属于 **Training-free（无需训练）** 的即插即用方法，因此不涉及训练时长，主要关注推理阶段的加速。

### 5. 实验数量与充分性
*   **实验规模**：作者在 4 种不同架构的模型上进行了广泛测试，涵盖了从 0.25 到 0.95 的多种剪枝比例。
*   **消融实验**：针对任务/视觉分支的保留比例、解耦编码策略、FCMA 对齐策略等关键组件做了详细的消融研究。
*   **公平性**：实验在相同的硬件和模型基座下进行，对比了多种最新的 SOTA 方法，并提供了 FLOPs 和实际运行时间的量化对比，实验设计较为客观、充分。

### 6. 主要结论与发现
*   **高性能与高压缩比**：在 LLaVA-1.5-7B 上，仅保留 11.1% 的视觉 Token（剪枝率 88.9%），仍能保持 98.1% 的原始性能。
*   **显著加速**：在 11.1% 的保留率下，实现了 **10 倍的 FLOPs 减少**，预填充（Prefill）阶段加速达 **2.3 倍**，整体推理加速 1.6 倍。
*   **超越 SOTA**：在多个基准测试中，DeSAP 在相同剪枝比例下的表现均优于现有的视觉中心或任务中心方法，尤其是在处理小物体和复杂语义对齐任务时优势明显。

### 7. 优点
*   **任务感知前置**：巧妙地在编码器端解决了 LLM 侧剪枝存在的空间偏见问题。
*   **计算开销极低**：解耦相似度的计算复杂度远低于 LLM 内部的跨模态注意力计算，且无需额外训练。
*   **鲁棒性强**：通过双源引导（任务+视觉），避免了因文本查询模糊而导致的过度剪枝。
*   **通用性好**：在 LLaVA 系列和 Qwen 系列模型上均表现出良好的泛化能力。

### 8. 不足与局限
*   **文本依赖性**：当文本查询非常简短或模糊（如“描述这张图片”）时，任务引导的效果会减弱，模型性能将更多取决于视觉显著性分支。
*   **长文本开销**：虽然视觉 Token 减少了，但如果输入文本极长，计算视觉-文本相似度的矩阵运算开销会增加（尽管文中提到已使用 CalibCLIP 策略缓解）。
*   **动态比例缺失**：目前任务和视觉分支采用固定的 50/50 分配比例，未来若能根据任务类型动态调整比例可能会进一步提升效果。
